# ROSIE and Help Center grounding corpus

This is the current operator and maintainer guide for the Riverside OS Help Center search index and ROSIE grounding sources.

The legacy `POST /api/ai/help`, `POST /api/ai/admin/reindex-docs`, `ai_doc_chunk`, and pgvector-era help tables were retired by migration 78. Do not use those paths for current release support.

## Current source of truth

| Artifact | Role |
|----------|------|
| `client/src/assets/docs/*-manual.md` | Bundled in-app Help Center manuals. |
| `client/src/assets/images/help/**` | Governed screenshots used by in-app manuals. |
| `client/src/lib/help/help-manifest.generated.ts` | Client-side Help manual manifest generated by `npm run generate:help`. |
| `server/src/logic/help_corpus_manuals.generated.rs` | Server-side Help corpus path list generated by `npm run generate:help`. |
| `docs/staff/CORPUS.manifest.json` | Approved staff procedure and operational docs for ROSIE/staff knowledge grounding. |
| `docs/ROSIE_OPERATING_CONTRACT.md` | Runtime ROSIE safety and source-of-truth contract. |
| `docs/HELP_CENTER_AUTOMATION.md` | Help screenshot and manifest refresh workflow. |

## Runtime retrieval

Current Help search uses the `ros_help` Meilisearch index and `GET /api/help/search`.

ROSIE uses the same Help retrieval path through the server-validated `help_search` tool context. It may also use approved read tools and live Store SOP when authenticated. It must not query raw business tables or use retired `/api/ai/*` routes.

Source priority for answers:

1. Server tool JSON for live facts and numbers.
2. Store SOP returned by `GET /api/staff/store-sop`.
3. In-app Help manuals and `docs/staff/*` procedure docs.
4. ROSIE contract docs and reporting/API catalogs.
5. Model prose, only as explanation of the sources above.

## Regeneration workflow

After changing in-app Help manuals or screenshots:

```bash
npm run generate:help
```

This regenerates:

- `client/src/assets/docs/help-quality-report.generated.json`
- `client/src/lib/help/help-manifest.generated.ts`
- `server/src/logic/help_corpus_manuals.generated.rs`

For the full Help Center refresh path, including AIDocs and Playwright screenshots:

```bash
npm run generate:help:refresh
```

To refresh the live Help search index while the API and Meilisearch are running:

```bash
npm run generate:help:refresh -- --reindex-search
```

or:

```bash
bash scripts/ros-ai-reindex-local.sh
```

The compatibility script now posts to `POST /api/help/admin/ops/reindex-search` with `full_reindex_fallback` enabled. The old script name is retained only for operator familiarity.

## Staff corpus validation

After changing `docs/staff/*.md` or `docs/staff/CORPUS.manifest.json`, run:

```bash
npm run help:aidocs:check
python3 scripts/verify_ai_knowledge_drift.py
```

The drift check verifies that manifest paths exist and staff guides are not accidentally left out of the approved corpus.

## Search and ROSIE quality checks

Use realistic staff queries when certifying the corpus:

- refund
- exchange
- close register
- blocked checkout
- offline recovery
- QBO failed
- RMS payment
- wedding pickup
- alteration ready
- receiving interrupted
- inventory adjustment
- loyalty points
- gift card refund
- split tender
- printer issue
- customer merge
- reprint receipt

Expected behavior:

- Help search should return the most operational manual first, not retired implementation notes.
- Recovery and escalation docs should be discoverable.
- QBO, RMS, close/register, payment/refund, receiving, physical inventory, and pilot governance queries should have at least one current-release result.
- ROSIE answers should cite the Help manual or staff doc that supplied the workflow.

## Retired behavior

Do not revive these without a new architecture decision and migration plan:

- `ai_doc_chunk`
- pgvector help embeddings from the retired AI tables
- `POST /api/ai/help`
- `POST /api/ai/admin/reindex-docs`
- arbitrary SQL generated from model output
- production customer/order/payment exports as learning input

If a doc, script, or answer still points staff to one of those paths, treat it as stale Help/ROSIE guidance and update it before release.

# Pilot observation checklist

**Audience:** Pilot lead, manager on duty, owner.

**Purpose:** Watch real staff behavior, not just whether screens technically work.

---

## How to observe

Stand back and watch the workflow. Do not coach unless the customer or money is at risk. Record what staff do naturally.

---

## What to watch

| Observation | Seen? | Notes |
|-------------|-------|-------|
| Staff hesitate before next action. | Yes / No |  |
| Staff ask where to click. | Yes / No |  |
| Staff ask what a status means. | Yes / No |  |
| Staff repeat a lookup for the same customer/member/item. | Yes / No |  |
| Staff abandon a workflow and start over. | Yes / No |  |
| Staff park or suspend work but forget to resume it. | Yes / No |  |
| Staff need manager override more than expected. | Yes / No |  |
| Staff use paper notes because screen state is unclear. | Yes / No |  |
| Customer waits while staff reconcile payment/balance/status. | Yes / No |  |
| Staff switch stations and lose context. | Yes / No |  |
| Staff recover cleanly after interruption. | Yes / No |  |
| Staff repeat the same recovery action. | Yes / No |  |
| Staff language shows confusion: "I think", "maybe", "not sure", "let me check again". | Yes / No |  |
| Staff avoid a feature and ask manager to do it. | Yes / No |  |

---

## Workflows to observe daily

- Checkout and receipt.
- Parked sale resume.
- Refund or exchange.
- Register close.
- Wedding pickup.
- Alteration intake and schedule.
- Receiving draft and post.
- Inventory lookup or adjustment.
- RMS charge or RMS review.
- QBO staging/review.

## End-of-day pattern check

Escalate to the issue log when:

- The same hesitation happens twice.
- Two staff ask the same question.
- A customer waits more than a normal checkout delay.
- A manager performs a workflow because staff are unsure.
- Any payment, refund, close, RMS, QBO, inventory, or garment release issue appears.

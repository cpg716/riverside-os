# Pilot issue log template

**Audience:** Manager on duty, pilot lead, accounting lead, and owner.

**Purpose:** One shared log for anything that slows, blocks, confuses, or risks the pilot.

---

## Severity definitions

| Severity | Meaning | Required response |
|----------|---------|-------------------|
| Informational | Not harmful, but useful to track. | Note during end-of-day review. |
| Watch closely | Repeated question, hesitation, or minor slowdown. | Review daily and coach staff if repeated. |
| Operational concern | Customer-facing delay, manager intervention, unclear state, or manual workaround. | Manager owns follow-up before next shift. |
| Pilot blocker | Workflow cannot be trusted without supervision or workaround. | Stop unsupervised use of that workflow and escalate same day. |
| Rollback-triggering | Payment, close, inventory, QBO, RMS, or customer-release risk that cannot be reconciled. | Halt affected workflow and start rollback decision review. |

## Reproduction expectations

For every issue, capture:

1. Where staff started.
2. The customer or work item involved.
3. The button or action staff chose.
4. The message, delay, missing information, or confusing state.
5. What recovery path was used.

Do not ask staff to repeat a payment, refund, inventory post, QBO approval, RMS reporting step, or register close just to reproduce an issue. Manager decides whether reproduction is safe.

## Escalation owner

| Issue type | Owner |
|------------|-------|
| Checkout, refund, parked sale, close warning | Manager on duty |
| RMS account charge or reconciliation | Manager on duty plus RMS/accounting owner |
| QBO staging, posting, mapping, reconciliation | Accounting owner |
| Wedding pickup, wrong member, balance confusion | Wedding lead plus manager |
| Alteration status, due date, ticket handoff | Alterations lead plus manager |
| Receiving, physical count, adjustment | Inventory lead plus manager |
| Printer, scanner, terminal, network | Technical owner plus manager |
| Staff confusion or repeated question | Pilot lead |

---

## Daily review status

Use one:

- **Open:** no owner or action yet.
- **Assigned:** owner and next action are clear.
- **Resolved:** no further pilot action needed.
- **Monitor tomorrow:** resolved enough to continue, but should be watched next shift.

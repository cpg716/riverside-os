# Riverside OS GO-LIVE Transaction Completion Matrix

Date: 2026-06-30

Scope: POS/register transaction completion paths that can create, pay, refund, exchange, pick up, or otherwise financially settle a Riverside OS transaction.

Recommendation: CAUTION until the listed Playwright lanes pass against the exact go-live commit. Code review found one audit-field persistence bug and no intentional bypasses of financial, inventory, tax, register-session, or permission protections in the reviewed checkout path.

## Completion Spine

The primary sale path is:

1. Register action in `client/src/components/pos/Cart.tsx`.
2. Tender entry and final button in `client/src/components/pos/NexoCheckoutDrawer.tsx`.
3. Payload construction, session pre-check, idempotency, offline/recovery handling in `client/src/hooks/useCartCheckout.ts`.
4. `POST /api/transactions/checkout` in `server/src/api/transactions.rs`.
5. `execute_checkout` in `server/src/logic/transaction_checkout.rs`.
6. Transactional writes to `transactions`, `transaction_lines`, `payment_transactions`, `payment_allocations`, inventory movement tables, gift-card/deposit/store-credit ledgers, fulfillment orders, wedding member state, shipment registry, alteration orders, and post-commit notification/activity systems.

Receipt printing is post-completion in `client/src/components/pos/ReceiptSummaryModal.tsx`; failures enqueue/recover print work and do not undo the financial transaction.

## Blocker Fixed In This Review

`client/src/hooks/useCartCheckout.ts` dropped explicit zero values for `rounding_adjustment` and `final_cash_due` because payload fields used truthy checks. That meant a valid `$0.00` cash-rounding audit value could be omitted even though the cashier intentionally recorded the zero final cash due state. The checkout transaction still completed, but the audit artifact was weaker.

Fix: preserve all non-nullish cent values, including `0`, via `optionalCentsField`. The Playwright tender contract now sends `final_cash_due: "0.00"` and verifies it persists through the transaction artifact endpoint.

## Remaining GO-LIVE Risk

The reviewed code path supports valid transaction completion without bypassing financial protections. Remaining risk is validation freshness, not an identified design blocker: the full required Playwright lanes and targeted Rust/client checks must pass against the exact release commit and database fixture set before declaring a final GO.

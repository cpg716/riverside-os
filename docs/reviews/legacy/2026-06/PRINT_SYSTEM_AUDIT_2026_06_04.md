# Print System Audit — 2026-06-04

## Scope

Reviewed the ROS print paths for receipt, tag, report, register/Z-report, customer, staff, help, inventory, order-list, and wedding print surfaces.

## Current Architecture

- **Receipt and cash drawer printing** uses `client/src/lib/printerBridge.ts` and routes to either Tauri native printing or `POST /api/hardware/print`.
- **Raw ESC/POS receipts** use `printRawEscPosBase64`, preserving ReceiptLine-generated bytes for Epson-compatible station printers.
- **Inventory tags** use `autoRoutePrint("tag", ..., "zpl")`, then fall back to a browser print preview if the direct Zebra path fails.
- **Full-page reports** mostly use browser print windows, with Z-report helpers exporting/opening HTML from the desktop app when available.
- **Failed receipt prints** are stored in the local retry queue so the completed sale is not blocked by printer downtime.

## Fixes Applied

- Tag and report station network ports are now read from the same localStorage keys that installer/server config sync already uses.
- Printer bridge network targets now trim addresses and reject empty addresses or invalid ports before dispatch.
- Server hardware print dispatch now rejects empty printer addresses just like printer readiness checks.
- Browser report/help/loyalty/scheduler print windows now use a shared load-safe print helper instead of timed `window.print()` fallbacks.
- Legacy wedding-manager order, appointment, party, and checklist print actions now use the same shared load-safe print helper.
- `/api/hardware/print` now enforces the saved `pos_station_config.printer_config` network-printer allowlist before dispatching to LAN targets.
- POS register sessions can now sync `printer_config` for their own active register lane, so browser/PWA print allowlisting no longer depends on Back Office admin sync.
- Failed receipt retry jobs now prune after 7 days, after 5 retry attempts, and above the 50 newest jobs.
- Hardware docs now use the real raw ESC/POS receipt API, `printRawEscPosBase64`, instead of the stale `autoRoutePrint(..., "raw_escpos_base64")` example.

## Remaining Risks

- None identified in the audited print paths.

### 2. Financial Calculations and Tax Logic

**Status:** IN PROGRESS

**Components Reviewed:**
- `server/src/logic/tax.rs` - NYS Publication 718-C implementation
- Tax calculation in `transaction_checkout.rs`
- Tax category resolution

**Key Observations:**

1. **Tax Logic (`server/src/logic/tax.rs`)**
   - Implements NYS Publication 718-C for Erie County clothing/footwear exemption
   - Exemption criteria: (Category: Clothing OR Footwear) AND (Net Price < $110.00)
   - When exempt: 4.00% State Tax = 0; 4.75% Local Tax active (4.75% combined)
   - Otherwise: 4.00% State + 4.75% Local (8.75% combined)
   - Tax categories: Clothing, Footwear, Accessory, Service, Other
   - Services are non-taxable (0% state, 0% local)
   - Rounding: Half away from zero (standard for USD)

2. **Tax Calculation in Checkout (`server/src/logic/transaction_checkout.rs`)**
   - Uses `nys_state_tax_usd` and `erie_local_tax_usd` functions
   - Calculates tax per line based on tax category and unit price
   - Validates tax values are not zero when expected
   - Handles tax exemption scenarios
   - Stores state_tax and local_tax separately on transaction_lines

3. **Tax Category Resolution**
   - Resolved from variant inventory data
   - Mapped from database text values
   - Used to determine applicable tax rates

**Potential Issues:**
- Tax logic appears to be correctly implemented according to NYS 718-C
- Rounding strategy is appropriate for USD
- Tax exemption threshold is correctly applied (strictly less than $110.00)

**Questions for Investigation:**
- Are tax categories correctly assigned to all products in the catalog?
- What happens if a product has no tax category assigned?
- Are there edge cases where tax calculation could be incorrect (e.g., very small amounts, very large amounts)?
- Does the system handle tax-exempt customers correctly?
- Are there any rounding issues that could cause penny discrepancies?

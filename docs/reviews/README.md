# Review And Audit Evidence

Status: **Historical review and audit evidence index**. Files in this folder are dated snapshots, not canonical current documentation. Use them for context, release evidence, and follow-up investigation after checking the current domain or integration front door in [../README.md](../README.md).

## Authority Rule

Review files can mention old bugs, migration ceilings, feature status, route names, or roadmap assumptions that were true at the time of review. They do **not** override canonical docs.

When a review conflicts with current documentation:

1. Use the current front door in [../README.md](../README.md).
2. Use the review file as evidence of what was observed on its date.
3. Update the canonical doc if the review reveals a still-valid gap.
4. Leave the review file historically intact unless it contains a broken link or misleading missing status banner.

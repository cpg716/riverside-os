# API Risk Register

Status: prioritized from initial high-risk audit pass.

| Priority | Domain | Risk | Rating | Evidence | Recommended Follow-Up |
|---|---|---|---|---|---|
| P0 | POS checkout | Duplicate or partially recovered checkout could create incorrect payments, allocations, inventory reservations, or Daily QBO Staging evidence if idempotency is incomplete. Full transaction checkout tests now pass in the server suite, and a lane-collision test fixture was hardened. | Critical | `POST /api/transactions/checkout`, `checkoutClientId`, provider recovery paths | Add duplicate/retry checkout endpoint tests with terminal success/failure/webhook recovery. |
| P0 | Refunds/voids/exchanges | Refund, void, and exchange endpoints mutate financial state and need strict idempotency, audit, and manager approval evidence. | Critical | `/api/transactions/{id}/refunds/process`, `/void`, `/exchange-settlement`, `/returns` | Add endpoint tests for auth, duplicate submit, manager override window, and audit rows. |
| P0 | Register close/cash adjustments | Z-close and drawer adjustments are direct cash reconciliation writes. | Critical | `/api/sessions/{id}/close`, `/begin-reconcile`, `/adjustments` | Add close-session permission/idempotency/audit tests. |
| P0 | QBO mapping/staging/sync | Incorrect mapping, approval, sync, retry, or void can post wrong accounting entries. | Critical | `/api/qbo/mappings`, `/granular-mappings`, `/staging/*` | Add permission and duplicate-click tests; expand journal basis tests. |
| P0 | Helcim payment/webhook | Webhook recovery or terminal attempt replay can finalize or alter payment state. Card refund/reverse duplicate client idempotency keys now reuse scoped attempts before another provider call. | Critical | `/api/payments/providers/helcim/*`, `/api/webhooks/card-events` (`/api/webhooks/helcim` legacy alias) | Add duplicate webhook replay, card refund/reverse duplicate-submit, and terminal attempt state-machine tests. |
| P0 | Inventory quantity/cost | Stock adjust, receiving, physical inventory publish, and pricing/cost updates directly alter inventory and margin. Product/pricing/procurement tests now pass in the full parallel server suite. | Critical | `/api/products/variants/*`, `/api/purchase-orders/*/receive`, `/api/inventory/physical/*/publish` | Add idempotency, audit, and RBAC tests for stock/cost writes. |
| P0 | Staff RBAC/PIN | Staff permission, role, PIN, and discount cap mutations authorize every other high-risk domain. | Critical | `/api/staff/effective-permissions`, `/admin/*/permissions`, `/set-pin` | Add central staff RBAC and audit tests. |
| P0 | Wedding financial links | Attach-order/cutover/group pay links can misattribute balances and fulfillment readiness. | Critical | `/api/weddings/attach-order`, `/cutover/links`, checkout `wedding_disbursements` | Add group pay/cutover/attach-order tests. |
| P1 | Customer merge/store credit/RMS | Customer identity and account balance mutations can corrupt financial/customer history. | Critical | `/api/customers/merge`, `/store-credit/adjust`, `/customers/rms-charge/*` | Add merge/store-credit/RMS mutation tests with audit assertions. |
| P1 | Shipping label purchase/webhook | Label purchase and webhook status updates affect fulfillment and external costs. | Critical | `/api/shipments/{id}/purchase-label`, `/api/webhooks/shippo` | Add duplicate label purchase and duplicate webhook tests. |
| P1 | Reporting basis drift | Booked vs fulfilled date errors can produce wrong financial/tax/commission reports. Schema repair added `transaction_lines.discount_amount`, which reporting and Customer Hub net-sales queries require. | Critical | `/api/insights/*`, QBO staging drilldowns | Add reporting contract tests for basis and timezone. |
| P1 | ROSIE permission preservation | AI tool-context or reporting routes could expose data beyond staff permissions. `capabilities` access mismatch is fixed and regression-tested. | High | `/api/help/rosie/v1/tool-context`, insight/product tests | Expand ROSIE permission matrix tests. |
| P1 | Public storefront payments/shipping | Public checkout and shipping routes must remain estimate/session-bound and idempotent. | High | `/api/store/checkout/*`, `/api/store/shipping/rates`, HelcimPay paths | Audit storefront checkout in a dedicated pass. |
| P1 | Counterpoint import replay | M2M import routes can alter customers, catalog, inventory, open docs, liabilities, and historical financial records. | Critical | `/api/sync/counterpoint/*`, `/api/settings/counterpoint-sync/*` | Add replay/idempotency tests and natural-key documentation per entity. |
| P2 | Provider credential storage | QBO/Helcim/Shippo credentials must never leak and must require admin settings gates. | High | QBO credentials, Helcim config, Shippo settings/webhook | Add redaction tests for status/config responses. |
| P2 | POS session fallback boundaries | POS session fallback is useful but can accidentally bypass Back Office-only permissions. | High | `require_staff_perm_or_pos_session`, customer/catalog/inventory reads | Add negative tests for POS session on Back Office-only mutations. |
| P2 | Audit log completeness | Several sensitive paths log actions, but completeness is not proven across all domains. | High | `staff_access_log`, domain event tables | Build audit-trail matrix per endpoint. |

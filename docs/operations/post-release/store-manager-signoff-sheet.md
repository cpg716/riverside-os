# Post-Release Store Manager Signoff Sheet

Use this sheet after store staff complete acceptance and smoke validation. The store manager should sign only when staff can operate RiversideOS safely in normal store conditions and know when to escalate.

| Signoff Area | Expected Result | Pass/Fail | Evidence Captured | Notes | Manager Signoff |
| --- | --- | --- | --- | --- | --- |
| Staff can complete core workflows | Staff can open register, complete sales, print/reprint receipts, handle pickup/balance due, search customers, and close register. |  |  |  |  |
| Staff can recover from failures safely | Staff know what to do when payment, printer, scanner, workstation, or provider state does not look correct. |  |  |  |  |
| Receipts are acceptable | Sale, refund, and reprint receipts are readable and show acceptable customer, payment, tax, and balance information. |  |  |  |  |
| Reports are understandable | Manager can review register, sales, refund, and accounting-facing reports without implementation staff explaining every field. |  |  |  |  |
| Help/diagnostics are usable | Staff can open Help, print Help if needed, and export diagnostics for support. |  |  |  |  |
| Escalation path is clear | Staff know which issues stop rollout, who owns escalation, and what evidence must be captured. |  |  |  |  |
| Manager can explain expected system behavior | Manager can explain expected behavior for checkout, refunds, register close, balances due, receipts, and QBO/accounting review. |  |  |  |  |

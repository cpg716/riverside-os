# Post-Release Operational Smoke Checklist

Use this checklist after installation or update and before store rollout. Critical checks must pass before deployment approval unless the owner records a written exception.

## Critical

| Workflow | Required Before Deployment/Store Rollout | Pass/Fail | Notes |
| --- | --- | --- | --- |
| Checkout | Complete cash, card, and split-tender sales with correct totals, tax, tender records, receipts, and balance due. |  |  |
| Refunds | Complete refund workflow with correct refund amount, provider evidence when applicable, and updated transaction/customer balance. |  |  |
| QBO | Generate QBO proposal and confirm Riverside totals, refund handling, liabilities, and exceptions are reviewable before approval. |  |  |
| Register close | Close register with cash, card, refund, and variance evidence captured for manager/accounting review. |  |  |
| Inventory receive | Receive purchase order items and confirm received quantity, exceptions, and item lookup behavior are usable by staff. |  |  |

## High

| Workflow | Required Before Deployment/Store Rollout | Pass/Fail | Notes |
| --- | --- | --- | --- |
| Help system | Staff can open and print Help content for the workflows they need during rollout. |  |  |
| Reports | Store reports open, show understandable totals, and can be reviewed or printed for manager/accounting use. |  |  |
| Diagnostics | Diagnostics export completes and produces evidence that support can use without asking staff to recreate the issue. |  |  |
| Updater | Workstation update check/install path is understood, version evidence is captured, and app reopens cleanly after update. |  |  |

## Informational

| Workflow | Required Before Deployment/Store Rollout | Pass/Fail | Notes |
| --- | --- | --- | --- |
| Copy polish | Staff-facing wording is clear, avoids internal jargon, and does not confuse transaction, fulfillment, payment, or customer actions. |  |  |
| Spacing | Important buttons, totals, and status messages are readable on the intended workstation screens. |  |  |
| Labels | Register, payment, refund, customer, order, and receiving labels match the language staff use during store work. |  |  |
| Animations | Motion or loading states do not hide required actions, totals, payment status, or error messages. |  |  |

# Post-Release Accounting Signoff Sheet

Use this sheet to confirm that RiversideOS rollout evidence is acceptable for accounting review. Attach register close, report, receipt, provider, and QBO evidence where applicable.

| Signoff Area | Expected Evidence | Pass/Fail | Evidence Captured | Notes | Accounting Signoff |
| --- | --- | --- | --- | --- | --- |
| Register totals | Register close totals match reviewed Riverside reports for the same register and date/range. |  |  |  |  |
| Cash totals | Cash expected, counted, and variance amounts are recorded and explained. |  |  |  |  |
| Card totals | Card sales/refunds match Helcim/provider evidence and Riverside payment records. |  |  |  |  |
| Refunds | Refund amounts, original transactions, provider evidence, and customer balance changes are reviewable. |  |  |  |  |
| Split tender | Each tender portion is recorded correctly and the transaction reaches zero balance only after all tenders are complete. |  |  |  |  |
| QBO proposal balance | QBO proposal balances to Riverside reports and exceptions are visible before approval. |  |  |  |  |
| Refund liability clearing | Refund liability entries are visible, explainable, and cleared or carried according to accounting direction. |  |  |  |  |
| Gift card liability | Gift card issued, redeemed, and remaining liability totals are reviewable. |  |  |  |  |
| Store credit liability | Store credit issued, redeemed, and remaining liability totals are reviewable. |  |  |  |  |
| Open deposit liability | Open deposits are visible, tied to the correct customer/order context, and not confused with fulfilled revenue. |  |  |  |  |

# Post-Release Store Acceptance Checklist

Use this checklist during store acceptance after a RiversideOS release is installed. Each workflow must be completed on the intended store workstation before rollout approval.

Record evidence that can be reviewed later, such as receipt numbers, transaction IDs, report exports, screenshots, printed pages, or manager initials.

| Workflow | Expected Result | Pass/Fail | Evidence Captured | Notes | Owner/Signoff |
| --- | --- | --- | --- | --- | --- |
| Register open | Register opens under the correct staff member, cash starting amount is recorded, and the open register is visible for sales. |  |  |  |  |
| Cash sale | Cash checkout completes, payment total matches amount tendered, change due is correct, and the transaction is available for review. |  |  |  |  |
| Card sale | Helcim card payment completes once, the approved payment amount matches the sale balance due, and the transaction shows paid. |  |  |  |  |
| Split tender | Sale accepts more than one tender type, each tender amount is recorded correctly, and the remaining balance reaches zero only after all tenders are complete. |  |  |  |  |
| Receipt print | Customer receipt prints clearly with correct store, transaction, tender, tax, and balance information. |  |  |  |  |
| Receipt reprint | Reprinted receipt matches the original transaction and is marked or understood as a reprint where applicable. |  |  |  |  |
| Refund | Refund completes for the correct transaction, refund amount is correct, payment/refund evidence is retained, and the customer balance is updated correctly. |  |  |  |  |
| Close register | Register close shows expected cash/card/refund totals, differences are explained, and closing evidence is saved. |  |  |  |  |
| Customer creation/search | New customer can be created, searched, selected, and used in a workflow without duplicate or missing customer details. |  |  |  |  |
| Existing order edit | Existing order can be opened, updated by authorized staff, saved, and reviewed with the expected customer/order details intact. |  |  |  |  |
| Pickup/balance due | Pickup workflow shows the correct balance due, accepts payment if needed, and records fulfillment without changing the wrong customer or transaction. |  |  |  |  |
| PO receiving | Purchase order receiving records the expected received quantity, updates receiving evidence, and leaves unmatched items clearly visible for follow-up. |  |  |  |  |
| Barcode scan | Barcode scanner enters the expected item or lookup value once per scan and does not add duplicate characters or wrong items. |  |  |  |  |
| QBO proposal generation | QBO proposal can be generated for the expected date/range, totals balance to Riverside reports, and exceptions are visible before approval. |  |  |  |  |
| Help print | Help content prints clearly from the intended workstation or printer and can be used by staff without implementation help. |  |  |  |  |
| Diagnostics export | Diagnostics export completes, produces a shareable support file, and includes the workstation/time context needed for support review. |  |  |  |  |

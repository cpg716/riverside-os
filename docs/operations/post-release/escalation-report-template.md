# Post-Release Escalation Report Template

Create an escalation report whenever a rollout issue could affect payment truth, refund truth, customer balances, register close, QBO balancing, hardware output, or staff ability to recover safely.

## Escalation Triggers

| Trigger | Escalate When | Immediate Evidence Required |
| --- | --- | --- |
| Accounting mismatch | Riverside totals do not match register close, payment provider, QBO proposal, or accounting review totals. | Register close, relevant report, payment/refund evidence, date range. |
| Duplicated payment | Customer appears charged more than once or Riverside records more than one payment for one intended approval. | Provider transaction ID, Riverside transaction ID, receipt, payment timeline. |
| Lost refund | Refund was expected but is missing from Riverside, provider evidence, customer balance, or accounting review. | Refund receipt/evidence, provider transaction ID if available, original transaction. |
| Incorrect balance due | Customer balance due is wrong before payment, after payment, after refund, or during pickup. | Transaction/customer record, receipt, payment/refund evidence, balance screenshot/export. |
| Printer corruption | Receipt, Help page, or report prints unreadable, partial, wrong customer, wrong totals, or repeated content. | Printed page/photo, workstation, printer model, transaction/report reference. |
| Stale provider state | Payment terminal, provider status, update state, or bridge status appears out of date after staff action. | Time observed, workstation, provider screen/status, Riverside screen/status. |
| QBO imbalance | QBO proposal does not balance or liability/refund lines do not match accounting expectations. | QBO proposal, Riverside report totals, register close, exception notes. |

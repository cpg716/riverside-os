# RiversideOS v0.4.5 — Store Cutover Readiness

## Overview

RiversideOS v0.4.5 is the release-prep build for Riverside's store cutover. It carries forward the v0.4 deployment topology:

- Backoffice / Server PC
- Register #1 Windows Tauri
- Register #2 iPad PWA
- Windows laptop PWA / optional Tauri clients

This release does not automatically mean production go-live is complete. It prepares the application, release metadata, and operator docs for the next GitHub release and Windows artifact pass.

## Key Improvements

### Store and Deployment
- Added online store workspace, merchandising, checkout, and operations support.
- Added the in-app update manager and Windows deployment package workflow support.
- Hardened Windows deployment provenance, installer behavior, and deployment manager setup.

### Payments and RMS
- Replaced legacy card payment operations with Helcim-centered payment operations.
- Added settlement visibility, webhook event logging, shared terminal routing, and secure credential settings.
- Added RMS account list snapshots and manual-first RMS Charge handling.

### Counterpoint Cutover
- Added cutover reconciliation visibility for customers, inventory, open docs, category/vendor mappings, gift cards, and loyalty.
- Added aggregate inventory fidelity checksums and targeted mismatch diagnostics.
- Kept gift cards and loyalty snapshot-only for cutover balances.

### Gift Cards
- Added promo gift cards for event/giveaway use.
- Promo cards support event names, one-year expiration, POS tender selection, and QBO expense mapping.
- Restored donated gift card visibility in the Gift Cards workspace navigation.

## Remaining Go-Live Gates

Before production install:

- Commit and tag `v0.4.5`.
- Push `main` and tag `v0.4.5`.
- Publish the GitHub release using `docs/releases/v0.4.5-github-release.md`.
- Run the Windows updater release workflow or produce a current manual Tauri bundle.
- Rerun GitHub checks on the final release commit.
- Complete physical station smoke on the Backoffice / Server PC, Register #1, Register #2 iPad PWA, and Windows laptop PWA surfaces.

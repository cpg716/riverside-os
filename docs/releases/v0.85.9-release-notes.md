# Riverside OS v0.85.9 Release Notes

Release date: 2026-06-04

## Summary

Riverside OS v0.85.9 is the current GO LIVE readiness release. It preserves the v0.85.0 POS Register hardening baseline and adds deployment hardening, role-aware in-app updates, standalone app self-updaters, ROSIE installer/runtime safety, Counterpoint Bridge GUI self-containment, customer join/split hardening, and Orders lifecycle workflow fixes.

## Added

- ROSIE AI installer automatic update detection using version marker files for Sherpa, STT, and TTS components.
- Standalone Tauri self-updater plumbing for Deployment Manager, ROS Server Manager, Counterpoint Bridge GUI, and ROS Dev Center.
- Release asset verification for POS and standalone updater manifests, including build metadata, build SHA, signatures, referenced artifacts, and matching `.sig` files.
- Orders lifecycle filters for Open Orders, All Records, Closed, Cancelled, NTBO, Ordered, Received, Needs Ready Check, Ready for Pickup, and Picked Up.
- Orders follow-up metric for received items that require staff ready-check review before customer pickup notification/release.

## Fixed

- Windows deployment connection probes now avoid terminating `NativeCommandError` failures during expected failed connection checks.
- Counterpoint Bridge GUI release builds are self-contained: packaged bridge resources and a bundled runtime replace customer-machine `npm install` and system `node` dependencies.
- New ROS-origin orders no longer show `CP Open Doc` or Counterpoint history labeling unless the transaction is a true Counterpoint import with Counterpoint document/ticket provenance.
- Customer join/split workflows preserve person-specific communications/profile views while keeping combined sales, history, and loyalty behavior clear for joined accounts.

## Changed

- Deployment Manager maintenance and repair layout now uses horizontal category tabs and a larger full-width output console.
- User-facing deployment labels now use Main Hub terminology instead of Server PC where appropriate.
- In-app update UX distinguishes Main Hub, Register, and Back Office update duties. Main Hub update copy covers app, server, ROSIE, and supporting services; workstation copy covers app/version-gate behavior.
- Updater checks use build metadata and build SHA values so same-version GitHub rebuilds are detected instead of only version-number bumps.
- ROSIE local llama.cpp profiles now support Intel i9-12900, Minisforum V3, Apple M3 Pro, Apple M3 Pro CPU-parity, and portable CPU testing. The Intel profile pins compute/batch threads to 8, disables GPU offload, enables mmap/mlock where supported, and applies P-core affinity on Windows.
- Orders status badges now summarize item lifecycle state instead of only transaction status, making Received, Needs Ready Check, Ready for Pickup, Closed, and Cancelled easier to read.

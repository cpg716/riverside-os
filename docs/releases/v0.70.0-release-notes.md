# Riverside OS v0.70.0 — Cash Rounding, Universal Search, ROSIE AI Copilot, and Host Commissioning

Riverside OS v0.70.0 is a major milestone release preparing the platform for live-retail operations. This release implements Sweden-style Cash Rounding, integrates the ROSIE AI Copilot and Universal Search Aggregator, stabilizes the core test suites, and provides production database cleanup scripts.

## What changed

### Cash Rounding & Consumer Financing
- **Sweden-Style Cash Rounding**: Cash transactions at POS checkout dynamically apply cash rounding to the nearest $0.05. Cash rounding offsets are recorded as a separate payment ledger entry (`cash_rounding_offset`) to ensure that base product prices, shipping, and tax lines are untouched and daily drawer reconciliation matches perfectly.
- **CoreCredit Financing**: Integrated consumer line-of-credit (CoreCard/CoreCredit) checks into the POS checkout payment allocations.

### ROSIE Local AI Copilot & Universal Search
- **ROSIE AI Copilot Integration**: Powered by a local Gemma LLM sidecar. Operates under the ROSIE Operating Contract: no direct SQL execution, full RBAC/permission validation, and user confirmations for data modifications.
- **Universal Search Aggregator**: Single aggregated backend endpoint `/api/search/aggregate` that queries Customers CRM, catalog inventory, alterations, active weddings, and help articles in a single call, replacing client-side multi-threaded lookups.
- **Transaction Backdating**: Added backdating of POS transactions (`booked_at` date overrides) adjusting payment allocations, commission calculations, and downstream QBO entries.
- **Dynamic Shortcuts**: Combined deterministic backend commands and dynamic Rosie AI search intents in the `GlobalCommandSearch` dialog without duplication.

### E2E Test Suite & Code Hygiene
- **Responsive & QBO Test Stabilization**: Scope locators in `pwa-responsive.spec.ts` to ensure 100% pass rate on responsive/PWA E2E tests.
- **Database Scrubbing**: Added `ros-wipe-business-data-keep-bootstrap-admin.sql` to safely purge all development/testing activity while leaving the seed/bootstrap system metadata and admin users intact.
- **Host Service Commissioning**: Documented and verified startup task integration (`install-server.ps1` registering Axum API as a Scheduled Task and PostgreSQL as an Automatic startup service).

## Deployment notes

- Backoffice / Server PC should use `http://127.0.0.1:3000` in the desktop app.
- Registers and mobile client PWAs should use the server's local LAN or Tailscale IP, e.g., `http://10.64.70.196:3000`.
- All deployment management should be executed via `RiversideOS-Deployment-Manager.exe` using version `0.70.0`.
- The local Gemma LLM model path defaults to `%LOCALAPPDATA%\riverside-os\rosie\` on Windows.

# RiversideOS v0.4.0 — Deployment Audit Release Candidate

## Overview
RiversideOS v0.4.0 is the deployment-audit release candidate for the production store rollout model:

- Backoffice / Server PC
- Register #1 Windows Tauri
- Register #2 iPad PWA
- Windows laptop PWA / optional Tauri clients

This release does not mean the store is automatically live. It makes the deployment status, station setup, release artifact requirements, and remaining go-live blockers explicit.

## Key Improvements

### Deployment Readiness
- Added current deployment status to the canonical store deployment guide.
- Documented the exact Windows installer/updater gap: `v0.4.0` assets must be produced before station install.
- Added first-time setup steps for the Backoffice / Server PC, including PostgreSQL, `FRONTEND_DIST`, firewall, migrations, and Shop Host smoke checks.

### Station Install Guidance
- Added role-specific install checklists for:
  - Register #1 Windows Tauri
  - Backoffice Windows app
  - Register #2 iPad PWA
  - Windows laptop PWA
- Clarified that Register #2 iPad PWA is a satellite lane and thermal receipts still require Register #1 or another Windows Tauri station today.

### Release Metadata
- Bumped package/app metadata to `0.4.0`.
- Updated release/go-no-go docs so missing `v0.4.0` Windows assets, GitHub rerun requirements, and physical station smoke remain visible.
- Fixed the local server Clippy blocker in the Meilisearch control-board helper.
- Fixed the individualized-shirt E2E fixture lookup so known custom SKU `200` uses the exact seeded catalog item.

## Remaining Go-Live Gates

Before production install:
- Publish/tag `v0.4.0`.
- Run the Windows updater release workflow or produce a current manual Tauri bundle.
- Rerun GitHub checks on the final release commit.
- Complete physical station smoke on the Backoffice / Server PC, Register #1, Register #2 iPad PWA, and Windows laptop PWA surfaces.
- Complete owner, operations, accounting, hardware, and backup/restore signoffs.

# Riverside OS v0.80.5 — Gold Release: In-App Database Self-Healing, POS Resiliency Hardening, and R2S Financial Account List Integration

Riverside OS v0.80.5 is our "Gold" Production Release, establishing complete parity between the Windows Deployment Manager and the In-App Updater through automated startup database migrations, while wrapping up major POS resilience updates, import engines, and security credentials scripting.

---

## What Changed

### 1. In-App Updater Database Self-Healing (Tauri-Deployment Parity)
* **Start-Up Database Migrations**: Enabled the `migrate` feature in the server's `sqlx` configuration and embedded the database migrations directory directly into the statically compiled server binary.
* **Self-Healing Schema**: The server now executes pending SQL migrations (`sqlx::migrate!("../migrations").run(&pool)`) dynamically on startup before performing schema contract checks. 
* **Deployment Parity**: When updated via the Tauri in-app updater (which updates the client desktop package containing the unified server), the application automatically updates the PostgreSQL schema to the latest version on launch—eliminating database schema mismatch crashes and matching the Windows Deployment Manager's migration behavior.

### 2. POS Hardening & Chaos-Regression Fixes
* **Asynchronous Post-Commit Handoff**: Decoupled non-critical back-office actions (such as commission logging and wedding event logging) from the critical transaction commit path. If these downstream databases/workers fail, they are caught and logged as non-blocking warnings, preventing a checkout failure from bricking the customer transaction.
* **Pre-Fetch Failures Visibility**: Surfaced silent pre-fetch failures (such as loading active store discounts or wedding purchase context) as user-friendly toast warnings for the cashier instead of failing silently or crashing.
* **Helcim Payment Circuit Breaker**: Introduced a sliding 60-second health monitoring window for the Helcim API. If three or more errors occur within 60 seconds, the circuit breaker trips, temporarily hiding credit card options and gracefully degrading the POS to Cash/Check checkouts. Cashiers are presented with a clear banner and a manual "Retry" action.
* **Offline Checkout Retry Queue**: Implemented a retry-once policy for checkout 5xx errors. Persistent connection errors automatically serialize and enqueue the transaction into the offline queue, allowing local checkout to continue operating without internet.

### 3. R2S Financial Account List Excel Import
* **Automated Workbook Parsing**: Created an import pipeline (`rms_account_list_import.rs`) using `calamine` to parse "AccountListReportData" Excel workbooks.
* **Validation & Checksums**: The parser validates report titles, merchant names, institutions, and timestamps. It calculates and compares total balance, minimum due, past due, and open-to-buy values against sheet footers, throwing validation warnings for sub-penny mismatches.
* **Transactional Integrity**: Inserts batches and account snapshots within a single Postgres transaction, guaranteeing database cleanliness on partial import failures.

### 4. Integration Credentials Sync Scripts
* **Cross-Platform Utility**: Developed PowerShell (`Export-IntegrationCredentials.ps1`, `Import-IntegrationCredentials.ps1`) and Bash (`export-integration-credentials.sh`, `import-integration-credentials.sh`, `sync-integration-credentials.sh`) scripts.
* **Durable & Secure Schema Sync**: Enables system operators to migrate or synchronize encrypted integration credentials (Helcim, QBO, Counterpoint bridge keys) between workstations securely via `pg_dump` and `psql` without writing raw secrets or credentials keys to logs or files.

### 5. CoreCard / CoreCredit Integration Removal
* **Codebase Purge**: Completely removed all references, configuration flags, E2E webhook tests, and setup scripts associated with the legacy CoreCard/CoreCredit store financing integration, simplifying POS checkout selections.

### 6. Non-Blocking Receipt Printer Diagnostics
* **Unblocked Offline Testing**: Changed physical receipt printer connectivity checks from a blocking barrier to a non-blocking UI warning. Developers and remote QA operators can now bypass physical printer checks and access the POS checkout/keypad interface.

---

## Deployment Notes

* **Workspace Version Parity**: All build manifests (`package.json`, Cargo workspaces, Tauri configurations, workflows, and deployment templates) are locked at version `0.80.5`.
* **Database Updates**: Database migrations are applied automatically on server launch. Hand-applying migrations via the command line is no longer strictly required for application updates.

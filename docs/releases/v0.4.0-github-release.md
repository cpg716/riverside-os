# RiversideOS v0.4.0 — Deployment Audit Release Candidate

RiversideOS v0.4.0 prepares the production deployment path for the store rollout model: Backoffice / Server PC, Register #1 Windows Tauri, Register #2 iPad PWA, and Windows laptop PWA/optional Tauri clients.

## Highlights

- Version metadata bumped to `0.4.0` across root, client, server, and Tauri manifests.
- Canonical deployment docs now include the current release artifact status and station setup sequence.
- Added Backoffice / Server PC setup steps for PostgreSQL, `riverside-server`, `FRONTEND_DIST`, firewall, migrations, and Shop Host smoke.
- Added install checklists for Register #1 Windows Tauri, Backoffice Windows app, Register #2 iPad PWA, and Windows laptop PWA.
- Go/no-go checklist now calls out the required `v0.4.0` Windows installer/updater assets before station install.

## Deployment Note

This release candidate still requires operational signoff before live store use. Before installing Windows stations, publish the `v0.4.0` release assets by running the Windows updater release workflow or producing a current manual Tauri bundle.

Known remaining gates: rerun GitHub checks on the final release commit, publish the Windows updater assets (`latest.json`, MSI, `.sig`), and complete the physical station smoke before live store use.

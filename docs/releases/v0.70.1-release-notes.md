# Riverside OS v0.70.1 — ROSIE E4B Model, Deployment Manager Diagnostics, Hotfix Cleanup

Riverside OS v0.70.1 is a deployment-quality patch release that standardizes the ROSIE AI model across the entire stack, significantly improves the Deployment Manager's diagnostic and lifecycle capabilities, and removes stale script copies that were a maintenance hazard.

## What changed

### ROSIE AI Model — Gemma 4 E4B

- Standardized on **Gemma 4 E4B** (4B params, 5.4 GB Q4_K_M GGUF) as the pinned production model, replacing E2B.
- Updated `MODEL_PIN.json`, Rust default paths, PowerShell installers, dev scripts, e2e mocks, documentation, and environment examples.
- Fresh installs, updates, and dev environments all resolve to the same E4B model automatically.

### Deployment Manager Improvements

- **PostgreSQL Status Panel**: Live diagnostics showing PG Windows service state, `psql` connectivity, PG version, database existence, size, table count, and migration count. Includes a Refresh button.
- **PostgreSQL Service Control**: Start PG / Restart PG / Stop PG buttons inside the status panel with auto-refresh after actions.
- **Stop Server**: Added a Stop Server button — previously only Start and Restart were available.
- **Uninstall Server**: Removes the server binary, client bundle, scheduled task (`Riverside OS Server`), and firewall rule. Preserves the PostgreSQL database.
- **Uninstall Register**: Removes the desktop POS app and shortcuts from the workstation.
- **Inventory tag print date**: Tag Designer footer text now automatically includes the print date on every inventory tag (HTML preview and Zebra/ZPL).

### Deployment Script Fixes

- Deployment Manager scripts receive `-ConfigPath`, run from the package root, and can relaunch elevated.
- Privileged actions are blocked with a clear message when not running as Administrator.
- Hardened config path resolution, `installRoot` defaults, null-safe package manifest checks, Postgres user normalization (`Admin` → `postgres` / `riverside_app`), and `ros_schema_migrations` audit probe.
- `apply-riverside-migrations.ps1`: Safe property updates when `server` or JWT fields are missing from saved config.

### Removed

- **Stale `hotfix/` directory**: Deleted ~7,800 lines of duplicated deployment scripts that had fallen behind the canonical `deployment/windows/` copies. All deployment scripts now live in a single location: `deployment/windows/`.

## Deployment notes

- All versions are `0.70.1` across root, client, server, Tauri, and Deployment Manager manifests.
- Backoffice / Server PC should use `http://127.0.0.1:3000` in the desktop app.
- Registers and mobile client PWAs should use the server's local LAN or Tailscale IP, e.g., `http://10.64.70.196:3000`.
- The ROSIE LLM model path defaults to `%LOCALAPPDATA%\riverside-os\rosie\models\gemma-4-e4b\` on Windows.
- Existing E2B model files can be removed after the E4B model is verified.

## Deployment Manager — Full Lifecycle

The Deployment Manager now covers the complete workstation lifecycle:

| Action | Description |
| --- | --- |
| **Install** (Wizard) | Step-by-step server or register installation |
| **Update Server** | Copy new binary, client files, run migrations, restart |
| **Update Register** | Install or update the desktop POS app |
| **Start / Stop / Restart Server** | Control the `Riverside OS Server` scheduled task |
| **PostgreSQL Status** | Live service state, connectivity, version, DB size, migrations |
| **Start / Stop / Restart PG** | Control the PostgreSQL Windows service |
| **Audit** | Run `audit-system.ps1` for full system health check |
| **Apply Migrations** | Run pending schema updates |
| **Seed Database** | Insert baseline data |
| **Start Fresh** | Factory reset (drop DB, migrate, seed) |
| **Repair** tools | Credentials key, admin account, Counterpoint bridge, ROSIE AI |
| **Uninstall Server** | Remove server files, task, firewall rule (keeps DB) |
| **Uninstall Register** | Remove desktop app and shortcuts |

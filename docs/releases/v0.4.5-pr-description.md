# RiversideOS v0.4.5 — Store Cutover Readiness

## Summary

This release-prep change bumps RiversideOS to `0.4.5` and prepares the repository for the next GitHub release. It aligns root app metadata, the POS/client package, the Rust server, the Tauri shell, lockfiles, deployment defaults, changelog, and release docs.

## What Changed

- Bumped application/package metadata to `0.4.5`.
- Updated current deployment docs and Windows package defaults from `v0.4.0` to `v0.4.5`.
- Added release notes, GitHub release copy, and PR description for `v0.4.5`.
- Documented that the latest published GitHub release is still `v0.4.0` until this release is tagged/published.
- Documented independent tool versions that are not part of the app semver bump.

## Verification

- Version bump command: `npm run bump -- 0.4.5`
- Root lockfile refreshed: `npm install --package-lock-only --ignore-scripts`
- Client lockfile refreshed: `npm --prefix client install --package-lock-only --ignore-scripts`
- Cargo lock metadata updated for server and Tauri package entries.

Release-prep validation completed:

- `cargo fmt --manifest-path server/Cargo.toml --check`
- `cargo metadata --manifest-path server/Cargo.toml --format-version 1 --no-deps`
- `cargo metadata --manifest-path client/src-tauri/Cargo.toml --format-version 1 --no-deps`
- `cargo check --manifest-path server/Cargo.toml`
- `npm ci --dry-run`
- `npm --prefix client ci --dry-run`
- `npm --prefix client run lint`
- `npm --prefix client run typecheck`
- `git diff --check`

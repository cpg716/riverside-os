# RiversideOS v0.3.0 — Operational Perfection

## Overview
RiversideOS v0.3.0 is a refinement release focused on clarity, trust, and operator efficiency. It strengthens the workflows staff already rely on every day, with targeted backend support where correctness required it.

## Key Improvements

### Inventory Truth
Inventory state is easier to understand from a single product view. Staff can quickly see what is on hand, reserved, available, and recently changed.

### System Timeline
Core history views are more readable and easier to investigate. Product, order, customer, and register history now better explain what happened and when.

### Guided Workflows
Receiving, exchanges, wedding lifecycle work, and register-close flows now provide clearer stages and next-step guidance. Staff no longer need to rely as heavily on memory.

### Error Prevention
Warnings appear earlier and explain risks more clearly. The system now helps staff avoid common mistakes using existing rules and validations.

### Operations Dashboard
The Operations overview now acts more like a command center. Staff can see what changed today, what needs attention, and which issues matter most in one place.

### Alterations Workbench
Alterations are now surfaced as garment work, not order work. The Alterations Hub highlights overdue work, due-today work, ready-for-pickup work, total open work, source labels, customer context, and searchable open alterations.

### Customer Lifecycle
Customer lifecycle is now explicit. Staff can quickly understand whether a customer is new, active, pending, pickup-ready, completed, or in an issue state.

### Customer Intake
Customer add/edit flows are more compact and easier to use at the register and Back Office. Duplicate review, address lookup feedback, and profile alteration visibility are clearer without blocking manual entry.

### Rule Visibility
Inventory, order, and charge rules are explained more clearly near the point of use. Staff can understand system behavior without relying on internal knowledge.

### Register Coordination
Multi-register visibility is clearer across active sessions, shared drawers, and pending close activity. Coordination risks are easier to spot before they cause confusion.

### Existing Order Payments
Register checkout now has the allocation foundation needed to collect payment for today’s merchandise and apply part of the same tender to existing open transactions without reopening or mutating existing order line items.

### QuickBooks Settings
QuickBooks credentials now live in Settings, while the bridge focuses on connection status and guidance instead of credential entry.

### Data Quality Signals
Catalog, customer, and order completeness gaps are now surfaced with lightweight indicators. Quality issues are easier to catch before they slow down real work.

### Decision Layer
Existing reporting data now includes clearer takeaways. Staff see what changed, what matters, and what needs attention without interpreting raw metrics alone.

## Operator Impact
- Fewer mistakes
- Faster workflows
- Clearer decisions
- Less reliance on training and memory

## What Did NOT Change
- No new standalone modules
- No architecture rewrites
- No alteration-only sales flow
- No weakening of checkout, customer save, or search failure behavior

## Technical Notes
This release reuses existing platform contracts and adds targeted migrations/API support for operational correctness, including alterations, payment allocation, search, and integration settings. Targeted test coverage was added or updated across the touched surfaces, and the release passed the main GitHub lint and Playwright E2E workflows.

## Final Statement
RiversideOS v0.3.0 moves the platform from feature-complete -> operator-perfect.

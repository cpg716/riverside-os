# Riverside OS v0.85.0 — GO LIVE Readiness & Production Hardening

## Overview
Riverside OS v0.85.0 is the GO LIVE readiness and production hardening release. It introduces critical checkout stability fixes, advanced security and privacy configurations, a local AI voice copilot upgrade, automated end-of-day breakage sweeps, a macOS companion app for developers, new staff tools, and operational dashboard refinements.

---

## 🚀 Key Feature Areas & Hardening

### 1. 🔧 POS Register Hardening & Stabilizations (Fixes A–F)
* **Fix A — POS Session Token Expiry Pre-Check**: Pre-validates session expiration before tendering to avoid late server rejections. `useCartCheckout` probes `GET /api/sessions/current` before tendering.
* **Fix B — Server-Side Printer Config**: Stores lane-specific printer configurations in `store_settings.pos_station_config` to eliminate local-only configuration drift.
* **Fix C — Blocked Offline Items Recovery**: Cart header polls the offline queue and displays amber ("syncing") / red ("recovery needed") badge states.
* **Fix D — Print Failure Retry Queue**: Captures failed receipt prints in a new `localforage` retry queue (`printRetryQueue.ts`) and adds a POS header button to open a print retry modal.
* **Fix E — Dynamic Register Lane Dropdown**: Generates register lane options dynamically based on `max_register_lanes` settings.
* **Fix F — Helcim Terminal Stream Auto-Reconnect**: Runs fallback polling alongside Helcim SSE stream to prevent stuck payment drawers.

### 2. 👥 Staff Profile, Personal Purchases & Dynamic Cart Pricing
* **Personal Purchases Grid**: Staff profiles (`Settings → Profile`) now display a paginated, searchable grid of personal purchases when linked to a CRM Customer record.
  - **Inline Receipt Viewer**: Clicking "View Receipt" launches the `ReceiptSummaryModal` directly from the profile page.
* **Dynamic Pricing Recalculations**: Rewrote the cart actions engine (`useCartActions`) to dynamically recalculate, track, and apply employee/customer discounts immediately when a profile is attached, changed, or detached.
* **Employee Cost-Plus & Discount Threshold Exemptions**: Ensures that employee pricing cascades automatically from catalog records without triggering manager discount threshold validation flags.
* **Cashier Code Collision Resolution**: Resolved potential unique constraint collisions when inserting test staff by verifying and allocating random unused 4-digit codes.

### 3. 🤖 ROSIE Local AI Copilot & Voice Stack (Phases 1–3)
* **Token Telemetry**: Tracks token usage and costs directly (`060_rosie_token_telemetry.sql`), allowing administrators to compare local LLM performance against cloud APIs.
* **Provider Abstraction (`RosieLLMProvider`)**: Allows runtime selection between local Gemma (via `llama-server`) and the Google Gemini API.
* **Smart Provider Selection Mode**: 
  - **LocalGemma**: Forces local Gemma for absolute privacy.
  - **GeminiApi**: Uses cloud Gemini API (with fallback to Local Gemma).
  - **Auto**: Prefer local Gemma for sensitive queries, and Gemini for help search, conversation, and heavy reasoning when keys are available.
* **Speech Stack (STT/TTS) Failover & Streaming**:
  - Dynamically shows `Whisper.cpp` (STT) when `SenseVoice` is unavailable.
  - Dynamically shows `Host Speech Command` (TTS) when `Kokoro` is unavailable.
  - Added streaming TTS using the `--stream` flag.
  - Custom UI redesign with glanceable runtime status cards and metric widgets.
* **Deployment Hardening**:
  - Pinned Python 3.12 and forced binary wheels for `sherpa-onnx` voice package installation.
  - Supports Windows `ProgramData` pathing for multi-user / SYSTEM server deployments.
  - Meilisearch help cap raised from 40 to 100 search hits.
* **E2E Testing & Gateway**:
  - Integrated a dedicated E2E API gateway (`e2e_gateway.rs`) for workflow testing.
  - Added Playwright workflow scripts (`rosie-e2e-workflows.mjs`) to test voice commands and queries.

### 4. 🖥️ macOS DevOps Center Companion App
* **Operations Dashboard**: Aligned the macOS DevOps status grid with the core ROS Operations grid to report system diagnostics.
* **Tailscale CORS Bypass**: Bypasses CORS boundaries when connecting to the local server over a Tailscale private VPN network.
* **Bug Manager / Developer Errors Integration**: Reorganized settings tabs and integrated the Developer Errors bug reports dashboard directly into the macOS DevOps app.
* **System Integrity Logs**: Added `ops_connectivity_logs` table (`061_ops_connectivity_logs.sql`) and `integration_heartbeat.rs` to run automatic health checks on:
  - QuickBooks Online (QBO)
  - Podium
  - Shippo
  - Weather
  - Fal.ai
  - NuORDER
  - Meilisearch

### 5. 💰 QuickBooks Online (QBO) Staging & Gift Card Breakage
* **Automated Expiration Breakage Sweep**: At the end of each business day, the system executes `sweep_expired_gift_cards` to audit active gift cards, write breakage event logs, deplete expired balances, and stage QBO journal entries.
* **Breakage Mapping**: Debits `liability_gift_card` (reduces liability) and credits `income_gift_card_breakage` (recognizes breakage revenue).
* **Staging Lifecycle Actions**: Support for reverting to pending, retrying failed, and voiding synced entries.

### 6. 🛡️ Order Pickup Guards & Zero-Balance Checkout
* **Strict Stock Availability**: Validates that stock-on-hand is greater than or equal to quantity for all unfulfilled lines.
* **Fulfillment Lifecycle Check**: Verifies that items transitioned to `ReadyForPickup` have been physically received (`received_at` is not null).
* **Zero-Balance Pickup Checkout**: Hardens `useCartCheckout` to process zero-dollar payments via POST `/api/transactions/{id}/pickup` when the balance due is zero.
* **Manager Override**: Bypasses stock/received validations using a manager PIN and a minimum 12-character description.

### 7. 🎨 Print Layouts, Formatting & Exporting
* **Z-Report & Daily Sales Print Overhaul**: Redesigned tables into high-density activity cards with payment pills, monospace pricing, and Tauri file save dialogs.
* **CSV Export with Totals**: Adds sum rows for transactions, sales, and tax totals, utilizing Tauri's native dialog fallback.
* **WiX Toolset Installer Retry**: Retries downloading packaging dependencies in CI/CD to prevent 502 connection timeouts.
* **Tax Assertion Alignment**: Realigned E2E tax audit assertions with the NYS tax audit response schema.

### 8. 💻 Countersync Desktop Sync Bridge GUI App
* **Single Standalone Installer**: The entire sync bridge—including background extraction scripts and dependency managers—is compiled and distributed as a single MSI/EXE installer.
* **Tauri Resource Bundling**: Script resources are packed directly into the application bundle, checking and resolving resource paths dynamically.
* **Automated Background Dependency Setup**: Automatically detects missing dependencies (`node_modules`) on launch and executes `npm install` cleanly in the background.
* **Live Process Log Console**: Streams real-time stdout and stderr from the background child extraction engine, color-coding successes, heartbeats, and database connectivity errors.
* **Interactive SQL Query Tester**: Exposes a safe environment to test configured sync mappings or custom raw SQL queries, returning a 10-row visual table preview before running exports.
* **Visual Status Control**: Start, stop, dry-run, and manual schema extraction runs controlled dynamically via a modern WowDash glassmorphism dashboard.

### 9. 📈 Operations Home Dashboard & Till Control Refinements
* **Till Control Terminology**: Renamed layout elements on the Operations Dashboard to use "Register Status" (top status indicator) and "Till Control" (right-hand control widget) instead of duplicate "Register Close" sections.
* **Running Cash Total**: Added a dynamic cash total indicator to the Till Control widget displaying `todaySummary.cash_collected` to give managers live cash metrics at a glance.
* **Layout & Modal Scroll Resets**: Tab switches in the operations center now reset the scroll position to the top. Integrated the `useDialogAccessibility` focus and scroll-to-top hook to the bug report and developer error detail modals.
* **Settings Tab Consolidation**: Removed the deprecated `ros-support-center` configuration routing, merging all support workflows under the unified `ros-operations-center` ("ROS Operations & Support Center").

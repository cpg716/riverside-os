# RiversideOS v0.3.1 — Production Hardening

RiversideOS v0.3.1 hardens the v0.3 release line for Hybrid Tauri Host retail validation. It adds audit evidence, release gates, recovery safeguards, and deterministic coverage across the highest-risk POS workflows.

## Highlights

- Checkout tender financial contracts for split tender allocation, check-number enforcement, cash rounding, and existing balance payment.
- Offline checkout recovery now keeps blocked replay rows and blocks register close until recovery is resolved.
- Register close now purges server-backed parked sales inside the close transaction with audit rows.
- Tax, inventory, commission, register, recovery, and QBO audit contracts are now part of release evidence.
- QBO staged journal approval/sync blocks unbalanced journals.
- QBO staging uses configured store-local business date, including near-midnight UTC coverage.
- Restore preflight and backup path guardrails are hardened.
- Production audit docs now clearly separate AI/code readiness from human/hardware/accounting signoffs.

# RiversideOS v0.4.5 — Store Cutover Readiness

RiversideOS v0.4.5 is the current release-prep build for the store rollout path. It builds on the v0.4 deployment foundation with the online store, Helcim payments operations, RMS Charge, Counterpoint cutover reconciliation, and promo gift card support.

## Highlights

- Version metadata bumped to `0.4.5` across root, client/POS, server, and Tauri manifests/lockfiles.
- Added online store workspace, merchandising, checkout, and operations flows.
- Added Helcim-centered payments operations, settlement visibility, webhook logging, shared terminal routing, and secure credential settings.
- Added RMS account list snapshots and manual-first RMS Charge handling.
- Hardened Counterpoint cutover readiness with customer, inventory, open-doc, category/vendor, gift card snapshot, loyalty snapshot, and inventory fidelity reconciliation.
- Added promo gift cards for events/giveaways with event names, one-year expiration, POS tender support, and QBO expense mapping.
- Stabilized local Playwright auto-boot and Back Office gift-card smoke coverage.

## Deployment Note

The latest published GitHub release before this prep is `v0.4.0`. Do not install Windows stations from `v0.4.5` until the matching Windows updater assets are published or an approved manual Windows package is produced.

Required Windows release assets remain: `latest.json`, Windows MSI, and `.sig`.

# Riverside OS v0.80.8 — Operational Clarity: Counterpoint Post-Sync Line Resolution

Riverside OS v0.80.8 is an operational clarity release focused on hardening the Counterpoint migration and sync workflows. It introduces post-sync line reconciliation to ensure that unresolved transaction and ticket lines map correctly once barcode and inventory catalog syncing completes, resolving discrepancies automatically and ensuring complete historical accuracy.

---

## What Changed

### 1. Counterpoint Post-Sync Line Reconciliation
* **Sweden-Style Fallback Mapping**: Unresolved transaction lines and ticket items now map cleanly to a fallback variant (`HIST-CP-FALLBACK`) with the original Counterpoint item key preserved in the `vendor_reference` column.
* **Database Reconciler**: Added a resolver (`resolve_unresolved_counterpoint_lines` in `server/src/logic/counterpoint_sync.rs`) that identifies unresolved items and dynamically heals them, mapping them back to correct catalog variants once their barcodes are loaded.
* **Receipt Reprinting & Returns Preservation**: This ensures historical receipts and tickets reprint with accurate product information, and returns or exchanges for legacy transactions resolve properly.

### 2. Proactive Dashboard Self-Healing
* **Dashboard Hook**: The post-sync resolver automatically executes whenever the Counterpoint Settings status dashboard API is loaded or refreshed.
* **Auto-Healing Warnings**: Operators do not need to run manual SQL queries. Fixing a barcode discrepancy and reloading the settings status page instantly updates and heals all corresponding historical references.

---

## Deployment Notes

* **Workspace Version Parity**: All build manifests (`package.json`, Cargo workspaces, Tauri configurations, workflows, and deployment templates) are locked at version `0.80.8`.
* **Database Updates**: Database migrations are automatically checked and applied on server launch. Run script `./scripts/apply-migrations-psql.sh` or double-click `Apply-RiversideMigrations.cmd` on Windows.

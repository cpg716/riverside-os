# Riverside OS v0.80.8 — Operational Clarity: Counterpoint Post-Sync Line Resolution

Riverside OS v0.80.8 is an operational clarity release focused on hardening the Counterpoint migration and sync workflows. It introduces post-sync line reconciliation to ensure that unresolved transaction and ticket lines map correctly once barcode and inventory catalog syncing completes, resolving discrepancies automatically and ensuring complete historical accuracy. This release also ships POS register security hardening (idle timeouts, shell containment), improved POS dashboard information cards, and receipt rendering fixes.

---

## What Changed

### 1. Counterpoint Post-Sync Line Reconciliation
* **Sweden-Style Fallback Mapping**: Unresolved transaction lines and ticket items now map cleanly to a fallback variant (`HIST-CP-FALLBACK`) with the original Counterpoint item key preserved in the `vendor_reference` column.
* **Database Reconciler**: Added a resolver (`resolve_unresolved_counterpoint_lines` in `server/src/logic/counterpoint_sync.rs`) that identifies unresolved items and dynamically heals them, mapping them back to correct catalog variants once their barcodes are loaded.
* **Receipt Reprinting & Returns Preservation**: This ensures historical receipts and tickets reprint with accurate product information, and returns or exchanges for legacy transactions resolve properly.

### 2. Proactive Dashboard Self-Healing
* **Dashboard Hook**: The post-sync resolver automatically executes whenever the Counterpoint Settings status dashboard API is loaded or refreshed.
* **Auto-Healing Warnings**: Operators do not need to run manual SQL queries. Fixing a barcode discrepancy and reloading the settings status page instantly updates and heals all corresponding historical references.

### 3. POS Register Security — Idle Timeout & Shell Containment
* **Register Idle Timeout (10 min)**: If the register is open and no interaction is detected for 10 minutes, the session is automatically cleared and the PIN login overlay reappears. The next cashier must re-authenticate before ringing a sale.
* **PIN Screen Idle Timeout (5 min)**: If the PIN login screen sits untouched for 5 minutes, the workstation navigates back to the POS Dashboard instead of sitting at an unattended login prompt.
* **Strict Shell Containment**: Closing the register (Z-report, session end) now always stays inside the POS Shell — the PIN overlay reappears instead of accidentally routing staff to the Back Office. Only the explicit "Back to Back Office" action exits POS mode.

### 4. POS Dashboard — Improved Information Cards
* **Today's Sales Card**: The "Register #N" stats card has been replaced with a live "Today's Sales" card showing the current day's booked sales total (net of returns), updated after each transaction.
* **Unread Notifications Card**: The "Priority Feed" stats card has been replaced with an "Unread Notifications" card showing the real-time unread count for the signed-in staff member. Clicking the card opens the Notifications Drawer directly.

### 5. Receipt Fixes
* **RMS Payment Line on Receipts**: RMS Charge Payment lines, gift card loads, and alteration service lines are no longer stripped from customer receipts. These lines represent real financial activity and now appear correctly on all receipt formats (HTML, thermal ESC/POS, email).
* **Receipt Text Wrapping**: Fixed "one or two letters dropping to a second line" on HTML receipts. Applied `table-layout: fixed`, `overflow-wrap: break-word`, and `width: 1%` constraints on the qty/price columns so the product name cell gets proper width and long names break cleanly at word boundaries.

### 6. Sales by Hour — Today-Only Data
* The POS Dashboard Sales by Hour snapshot card now explicitly filters to today's `business_date` before computing totals. Previously, if today had no hourly rows yet, the card would display the most recent prior day's total (e.g. $520 from the prior session) instead of $0.

### 7. Constant Contact Integration
* **Contact Marketing Sync**: Synchronizes opted-in customers (`marketing_email_opt_in == true`) to a default Constant Contact mailing list using high-performance bulk import activities.
* **Segment & Tag Mappings**: Map customer tags (e.g., `VIP`) or group codes (e.g., `Bridal`) to target mailing lists in Constant Contact.
* **Timeline Event Ingestion**: Real-time webhook tracking captures email events (sent, bounced, opened, clicked, unsubscribed) and logs them on the customer relationship timeline.
* **Security & Storage**: Client credentials and mapping configurations are stored encrypted at rest using `RIVERSIDE_CREDENTIALS_KEY`.

### 8. Inventory Pricing & Printing Hardening
* **Weighted Average Cost (WAC) Safety**: `server/src/logic/procurement.rs` clamps negative stock-on-hand to zero before WAC division, preventing divide-by-zero or nonsensical cost averages when receiving inventory against damaged/returned stock.
* **Backend Test Coverage for Pricing Patches**: Added 5 new integration tests in `server/src/api/products.rs`:
  - `patch_product_model_base_price_clears_shelf_labeled_for_base_variants_only` — confirms override variants are NOT touched when base price changes.
  - `patch_product_model_base_price_returns_out_of_stock_variants` — confirms out-of-stock base variants ARE returned in the response.
  - `patch_variant_pricing_detects_real_price_change` — confirms `price_changed: true` + `shelf_labeled_at` cleared on new override.
  - `patch_variant_pricing_no_op_when_effective_price_unchanged` — confirms `price_changed: false` + `shelf_labeled_at` preserved when resubmitting the same override.
  - `patch_variant_pricing_clear_override_detects_change` — confirms `price_changed: true` + `shelf_labeled_at` cleared when clearing an override that differed from base.
* **Browser Print Fallback Popup Blocker Detection**: `openInventoryTagsPreviewWindow` in `client/src/components/inventory/labelPrint.ts` now returns `"blocked"` when `window.open` returns `null` (e.g., popup blocker). The fallback chain propagates this status. All UI callers (`VariationsWorkspace`, `ProductHubDrawer`, `InventoryControlBoard`) now toast an explicit error instead of silently failing or claiming success.
* **Batch Price Update Reprint Prompt**: `VariationsWorkspace` batch price updates now collect per-variant pricing responses and, if any variants experienced a real effective-price change with positive stock, present a single confirmation modal to print updated tags for all affected variations at once. Previously, batch updates silently skipped reprint prompting entirely.

### 9. Tailscale Remote Access Security Hardening
* **Admin Gate on Tailscale Status**: `GET /api/settings/remote-access/status` now requires `settings.admin` permission, matching the connect and disconnect endpoints. Previously, any authenticated staff could read Tailscale peer info and DNS names.
* **CORS Production Hardening Documentation**: Added "Production CORS hardening" section to `REMOTE_ACCESS_USER_GUIDE.md` explaining that `RIVERSIDE_STRICT_PRODUCTION=true` and `RIVERSIDE_CORS_ORIGINS` must be set when deploying for Tailscale browser access. Without this, the server defaults to `allow_origin(Any)`, which widens the attack surface.

### 10. CI/CD — Windows Deployment Package Silent Upload Fix
* **PowerShell Error Handling**: `windows-deployment-package.yml` release upload step now sets `$ErrorActionPreference = "Stop"` and checks `$LASTEXITCODE` after every `gh release upload` command.
* **ZIP Existence Check**: The step now verifies the deployment ZIP exists with `Get-ChildItem` before attempting upload, catching glob/path mismatches early.
* **Impact**: Previously, a failed deployment ZIP upload was silently masked by a subsequent successful updater upload. The step appeared green while the ZIP was never attached to the release.

### 11. Counterpoint Bridge Category Display Names
* **IM_CATEG.DESCR Lookup**: `CP_CATEGORY_MASTERS_QUERY` in the Counterpoint bridge config now LEFT JOINs `IM_CATEG` and uses `COALESCE(c.DESCR, i.CATEG_COD)` for the `display_name` field. This produces human-readable category names in the Riverside catalog when Counterpoint category descriptions are populated.

## Deployment Notes

* **Workspace Version Parity**: All build manifests (`package.json`, Cargo workspaces, Tauri configurations, workflows, and deployment templates) are locked at version `0.80.8`.
* **Database Updates**: Includes Migration `048_constant_contact_integration.sql` to set up tables for sync logs and email tracking events. Database migrations are automatically checked and applied on server launch. Run script `./scripts/apply-migrations-psql.sh` or double-click `Apply-RiversideMigrations.cmd` on Windows.
* **CI Lint Fix**: Applied `cargo fmt` to `embedded_migrations.rs` (single-line tuple style) to resolve the GitHub Actions `cargo fmt --check` failure on the server-lint and tauri-lint jobs.
* **E2E Contract Testing**: Updated the Playwright `tender-matrix-contract.spec.ts` test suite to assert that the RMS payment line is printed on the receipt (changing `expect(receipt).not.toContain(...)` to `toContain(...)`), aligning test assertions with the new receipt requirements.

# ROS Release v0.50 GOLD

## Overview

ROS v0.50 GOLD is a launch-certification release for Riverside OS. This release is focused on stabilization, operational survivability, financial trust, lifecycle trust, workflow certification, visual verification, and Help Center asset readiness.

This release does not add broad new product scope. It hardens the existing store workflows and records the automated certification evidence needed before final production signoff.

## Certification Summary

Automated certification completed on 2026-05-14:

| Gate | Result |
| --- | --- |
| Standard Playwright release gate: `npm run test:e2e:release` | 310 passed, 31 skipped, 0 failed |
| Skipped-lane certification with E2E API/DB env and `E2E_RUN_VISUAL=1` | 31 passed, 0 skipped, 0 failed |

The 31 skipped tests in the standard release command were not accepted as uncertified. They were rerun explicitly with the E2E API/DB environment and visual mode enabled, and all 31 passed.

## Deployment Assets

The Windows deployment package workflow was run after the GOLD tag and release were published. It produced the review artifacts `riverside-windows-deployment-package` and `tauri-windows-updater-dist`.

The updater release workflow now accepts an explicit `release_tag` input. For ROS v0.50 GOLD, run `Tauri register updater release (Windows)` with `release_tag=v0.50-GOLD` so updater assets publish to the GOLD GitHub release instead of falling back to `v<client package version>`.

## Remaining Production Signoffs

The automated software certification is clean. Production rollout still requires the normal human/environment signoffs:

- Production or RC database audit probes.
- Hardware/register peripheral smoke checks.
- Backup and restore drill.
- QBO/accounting review.
- Owner and store operations approval.

## Release Decision

Automated software gate: **GO**.

Production rollout: **GO only after the remaining human/environment signoffs are complete or explicitly waived by ownership.**

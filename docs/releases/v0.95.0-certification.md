## Production update incident

On 2026-07-13, the published `86724e73` Windows package failed during a Main Hub update. The pre-migration dump used the limited `riverside_app` account and could not read the production-only `ros_repair_backup` schema. Rollback then restored an environment file whose application password no longer matched the PostgreSQL role, leaving the API offline until `DATABASE_URL` was synchronized from the protected deployment configuration. The restored v0.90.0 API returned HTTP 200 and the database remained intact.

The original `86724e73`, superseded `c3826bf6`, and prior `94f962c` Windows packages must not be reused. Published build `6411aa09` passed the release workflow, signed Windows/macOS publication, and package/updater provenance verification. Its update package explicitly preserves installed ROSIE LLM/STT/TTS assets, environment settings, processes, and scheduled task instead of invoking ROSIE setup. A successful Main Hub update/readiness check remains required for operational signoff.

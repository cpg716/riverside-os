# RiversideOS v0.3.1 — Production Hardening

## Summary
This release hardens RiversideOS v0.3 for production retail validation. It adds release-blocking audit contracts, closes AI-actionable P0/P1 production-readiness gaps, updates deployment documentation, and fixes CI-specific Playwright isolation issues found after the production hardening push.

## Key Changes
- Added production hardening audit outputs:
  - `docs/reviews/PRODUCTION_HARDENING_AUDIT_2026.md`
  - `docs/PRODUCTION_HARDENING_FIX_PLAN.md`
  - `docs/PRODUCTION_COVERAGE_GAP_MATRIX.md`
  - `docs/PRODUCTION_DEPLOYMENT_GO_NO_GO_CHECKLIST.md`
- Added release-blocking E2E audit contracts for checkout tender, tax, commission, inventory, offline recovery, QBO, and register close.
- Hardened offline checkout recovery and register close blockers.
- Hardened QBO approval/sync, business-date handling, token guardrails, and staged journal verification.
- Hardened restore preflight and backup path expectations.
- Stabilized POS RMS receipt and tax/QBO audit assertions for GitHub Actions data shape.
- Bumped package/app metadata to `0.3.1`.

## Go-Live Caveat
The software release is ready for final validation, but retail deployment remains blocked until the checklist signoffs are complete.

# RiversideOS v0.3.1 — Production Hardening

## Overview
RiversideOS v0.3.1 is the production-hardening release for the v0.3 line. It keeps the v0.3.0 operational workflow improvements and adds the audit evidence, release gates, and recovery safeguards needed before Hybrid Tauri Host retail validation.

This release is a software hardening release. Retail go-live still requires the human/environment signoffs listed in `docs/PRODUCTION_DEPLOYMENT_GO_NO_GO_CHECKLIST.md`.

## Key Improvements

### Financial Truth
- Added checkout tender financial contracts for check-number enforcement, split tender allocation, existing transaction balance payment, and cash rounding ledger/QBO impact.
- Hardened checkout recalculation and tender validation so financial artifacts remain traceable.

### Offline Recovery
- Offline checkout replay now retains 4xx failures as blocked recovery rows instead of deleting queued sales.
- Register close blocks while checkout recovery is pending or blocked.
- Staff/offline docs now describe blocked recovery handling.

### Register Close
- Z-close now purges server-backed parked sales inside the close transaction.
- Parked-sale purge creates per-sale audit rows.
- Register audit coverage verifies linked-lane lifecycle, closed-token rejection, and parked-sale purge behavior.

### Tax, Inventory, and Commission
- Added tax audit coverage for category inheritance, parent-product tax overrides, NYS/Erie clothing thresholds, discount crossing, stale client tax rejection, tax-exempt reason enforcement, gift-card non-taxability, shipping no-tax treatment, return tax reversal, and QBO tax liability mapping.
- Added inventory audit coverage for order-style no-decrement checkout, pickup decrement, duplicate PO receipt retry, and return/restock truth.
- Added commission audit coverage for fulfillment timing, rate specificity, finalized payout immutability, and internal SPIFF receipt exclusion.

### QuickBooks
- QBO approval/sync now blocks unbalanced staged journals.
- QBO proposal/drilldown date cuts use configured store-local business date.
- QBO token activation in strict production requires a non-default key, and new token writes use authenticated `v2:` wrapping.
- QBO audit coverage verifies balanced proposal, dedupe, drilldown linkage, one-time approval, and near-midnight business-date behavior.

### Backup and Restore
- Restore now has server-side preflight checks, backup catalog membership checks, strict-production guardrails, and local restore drill evidence.
- Strict production requires an explicit durable backup directory.

### Test and Release Evidence
- Full local release gate: `181 passed, 7 skipped, 0 failed`.
- Targeted CI-failure follow-up: POS RMS + tax audit specs reported `11 passed`.
- Release docs now separate AI/code gates from human/hardware/QBO/restore signoffs.

## Remaining Go-Live Gates

Before unattended retail deployment:
- Run SQL audit probes against RC/production data.
- Complete Hybrid Tauri host restore drill.
- Complete hardware/register station drill.
- Complete QBO/accounting signoff.
- Complete owner, operations, hardware, and backup/restore signoff.

## Final Statement
RiversideOS v0.3.1 is the software hardening release that makes the v0.3 line ready for final retail validation.

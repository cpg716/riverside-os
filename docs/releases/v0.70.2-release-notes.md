# Riverside OS v0.70.2 — PowerShell Deployment Hardening, Playwright E2E Stabilization, and CI Upgrades

Riverside OS v0.70.2 is a maintenance and deployment-hardening release. It resolves critical Windows PowerShell encoding incompatibilities that blocked installations, stabilizes the end-to-end (E2E) testing suite against redesigned UI components, and updates the CI/CD pipeline actions ahead of the Node.js 20 deprecation deadline.

## What changed

### 1. PowerShell Deployment Hardening (UTF-8 BOM Encoding)
* **Encoding Standardization**: Encoded all PowerShell deployment scripts (`deployment/windows/*.ps1`) with UTF-8 BOM.
* **ANSI Code Page Compatibility**: Standard Windows PowerShell 5.1 parses scripts using the system's local ANSI code page (e.g., Windows-1252/CP1252), causing syntax crashes on non-ASCII characters like the em-dash (`—`). By adding UTF-8 BOM and using ASCII equivalents, we've eliminated syntax errors such as `Unexpected token 'Could' in expression or statement` on target Windows registers.
* **Build Configuration**: Standardized default `$Version` to `"0.70.2"` in the deployment manager build packaging configurations.

### 2. Playwright E2E Test Suite Stabilization
* **Locator Alignment**: Refactored locators to align with recent POS shell redesigns:
  * Scoped the "mark sent" button locator to prevent matching incorrect rows in the direct invoice lifecycle.
  * Updated confirm button matches to correctly identify the new "Post Without Invoice" action.
  * Aligned the post receipt button locator with redesigned labels.
  * Handled stage-based workflow hints (rendering `Next: Post inventory` hint on the final/post stage).
* **Regex Consistency**: Anchored the print button regex match (using `/^print$/i` instead of `/print/i`) to ensure consistent, robust button locator targets across blocking test suites.

### 3. CI/CD Pipeline & Dependency Upgrades
* **GitHub Actions Upgrades**: Upgraded CI workflows to prevent Node.js 20 deprecation warnings:
  * Upgraded `actions/download-artifact` from `v4` to `v7`.
  * Upgraded `actions/github-script` from `v7` to `v8`.
* **Lockfile Synchronization**: Audited and bumped all lockfiles (`package-lock.json` in root, `client/`, and `deployment/manager-app/`) to ensure version alignment across the entire dependency graph.

### 4. Local Development & Setup Diagnostics
* **Native PostgreSQL Guide**: Documented instructions for developer setups to run directly against native PostgreSQL on port `5432` if Docker VM (OrbStack) environments freeze.
* **Startup Seeding**: Added clarity on the required baseline row insertion in the `store_settings` table (`id = 1`) to bypass the `RowNotFound` startup block.

## Deployment notes

* All versions are synchronized at `0.70.2` across root, client, server, Tauri, and Deployment Manager manifests.
* Windows deployment builds must be generated using `build-deployment-package.ps1` at the tag `v0.70.2`.
* Ensure that target servers have applied the active migration schema baseline using `apply-migrations-psql.sh` or the graphical update interface.

## Summary

RiversideOS v0.3.0 is a refinement release focused on operator clarity, trust, and day-to-day efficiency. It improves visibility, workflow guidance, and decision-readiness across the existing platform, with targeted backend support where correctness required it.

## Scope

This release covers all 10 operational-perfection areas:

1. Inventory Truth
2. System Timeline
3. Error Prevention
4. Guided Workflows
5. Operations Dashboard
6. Customer Lifecycle
7. Rule Visibility
8. Register Coordination
9. Data Quality Signals
10. Decision Layer

This is not broad module expansion. The work stays inside existing operational surfaces while adding targeted API/schema support for alterations, search, integration settings, and checkout allocation correctness.

## Commit Structure

1. `feat(inventory): unify inventory truth, receiving flow, and product timeline`
2. `feat(orders): improve transaction clarity and expose order and charge rules`
3. `feat(pos): surface receipt contact requirements before send`
4. `feat(customers): introduce lifecycle states and data quality indicators`
5. `feat(operations): add command center, decision summaries, and register coordination`
6. `feat(workflows): add guided step-based flows for key operations`
7. `docs: align manuals and help content with operational UI changes`

Additional cleanup:
- `chore(migrations): restore ledgered files and harden status check`
- `fix(ci): repair alterations selector coverage and SQLx-safe Meilisearch reindexing`
- `chore(release): prepare v0.3.0 metadata and changelog`

## Risk

- Moderate release surface because checkout allocation, alterations, search, and integration settings were touched
- No broad architecture rewrite
- No new standalone module
- Financial correctness remains the highest-risk review area

Main review focus:
- Checkout allocation behavior for existing order payments
- Alterations workbench/search/customer-profile visibility
- Customer intake duplicate/address behavior
- QuickBooks settings/bridge connection guidance

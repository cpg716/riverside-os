## Summary

RiversideOS v0.3.0 is a refinement release focused on operator clarity, trust, and day-to-day efficiency. It improves visibility, workflow guidance, and decision-readiness across the existing platform without introducing new modules, schema work, or backend expansion.

## Scope

This release covers all 10 operational-perfection areas:

1. Inventory Truth
2. System Timeline
3. Error Prevention
4. Guided Workflows
5. Operations Dashboard
6. Customer Lifecycle
7. Rule Visibility
8. Register Coordination
9. Data Quality Signals
10. Decision Layer

This is not feature expansion. The work stays inside existing contracts, APIs, data, and operator-facing surfaces.

## Commit Structure

1. `feat(inventory): unify inventory truth, receiving flow, and product timeline`
2. `feat(orders): improve transaction clarity and expose order and charge rules`
3. `feat(pos): surface receipt contact requirements before send`
4. `feat(customers): introduce lifecycle states and data quality indicators`
5. `feat(operations): add command center, decision summaries, and register coordination`
6. `feat(workflows): add guided step-based flows for key operations`
7. `docs: align manuals and help content with operational UI changes`

Additional cleanup:
- `chore(migrations): restore ledgered files and harden status check`

## Risk

- Low backend risk
- No schema changes
- No new backend systems
- Primarily UI, visibility, and workflow-guidance changes

Main review focus:
- The order surfaces are the largest diff in the set, especially the `OrdersWorkspace` / `TransactionDetailDrawer` grouping. That area is still in-scope, but it deserves the closest visual and behavioral review.

## Testing

- Targeted E2E coverage was added or updated across the touched surfaces
- Validation included lint, build, pack, server checks, and targeted E2E runs
- Local Playwright auto-boot had an environment-specific startup issue during validation
- Alternate-port targeted Playwright runs passed for the relevant changed surfaces

## Technical Notes

- Reuses existing APIs, existing server truth, and existing UI surfaces
- No new infrastructure

- Faster workflows

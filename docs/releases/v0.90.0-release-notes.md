# Riverside OS v0.90.0 Release Notes

Date: 2026-06-04

## Summary

Riverside OS v0.90.0 is the current source release for GO LIVE readiness. It preserves the v0.85.x POS, deployment, ROSIE, Counterpoint Bridge GUI, customer join/split, and Orders lifecycle baseline, then adds the final source-side cleanup needed before publishing fresh release assets.

## What Changed

- **QBO explicit mapping enforcement** — journal staging no longer uses a generic fallback account. Any exportable financial activity must have an explicit Chart of Accounts mapping before it can be posted.
- **QBO inventory adjustment key cleanup** — the legacy inventory adjustment revenue key was renamed to `REVENUE_INVENTORY_ADJUSTMENT` so the mapping matrix remains clear and auditable.
- **Pre-go-live local review fixes** — locally verifiable QBO, Counterpoint, backup/restore, Helcim, Podium, Shippo, and release/update code paths were reviewed, with targeted fixes applied where possible.
- **Helcim and Shippo test hardening** — health-check and credential-state tests now avoid nondeterministic local/CI behavior while still covering disabled, missing-token, and healthy states.
- **Docs/help refresh** — active deployment guides, manuals, Help Center sources, release notes, and certification evidence now point to v0.90.0 as the current release.
- **Release metadata bump** — root, client, server, Tauri, standalone tools, ROS Dev Center, and Windows deployment metadata are aligned at `0.90.0`.

## Release Assets

The v0.90.0 GitHub release must be published as Latest and must be populated by the release workflows after the tag is pushed.

Required assets:

- Windows POS updater manifest: `latest.json`
- Windows POS build manifest: `riverside-updater-build-manifest.json`
- Windows POS MSI and matching `.sig`
- Windows deployment package: `RiversideOS-v0.90.0-*-Windows-Deployment.zip`
- Standalone app packages for Deployment Manager, Server Manager, and Counterpoint Bridge GUI where produced by the Windows deployment workflow
- macOS ROS Dev Center DMG and updater assets where produced by the macOS workflow

## Validation

Source validation is tracked in [`v0.90.0-certification.md`](v0.90.0-certification.md). Release-asset validation is completed by the GitHub release workflows and final `gh release view v0.90.0` asset inspection after publication.

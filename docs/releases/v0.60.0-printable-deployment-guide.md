# Riverside OS v0.60.0 Printable Deployment Guide

Use this guide for the live-store install of:

- **Backoffice / Server PC**: PostgreSQL, Riverside server, web/PWA files, migrations, firewall, startup task, and optional Back Office desktop app.
- **Register #1 Windows PC**: Riverside desktop app, station API base, receipt printer, cash drawer, scanners, and primary till close.
- **Register #2 iPad**: PWA satellite register lane.
- **Other devices**: Back office browser/PWA workstations, phones, and support devices.

Print this guide and check each box during deployment.

---

## 1. Release Assets To Use

Use only the official GitHub release:

```text
https://github.com/cpg716/riverside-os/releases/tag/v0.60.0
```

Required assets:

- [ ] `RiversideOS-v0.60.0-af23cef-Windows-Deployment.zip`
- [ ] `Riverside.POS_0.60.0_x64_en-US.msi`
- [ ] `Riverside.POS_0.60.0_x64_en-US.msi.sig`
- [ ] `latest.json`
- [ ] `riverside-updater-build-manifest.json`

Expected release identity:

```text
Version: 0.60.0
Release SHA: af23cefccac60f60050e009c527987571a012510
Deployment package: RiversideOS-v0.60.0-af23cef-Windows-Deployment.zip
```

Do not mix files from older releases.

## 2. Store Deployment Roles

| Station | Device | Role |
| --- | --- | --- |
| Backoffice / Server PC | Windows PC | Runs PostgreSQL, Riverside server, web/PWA files, migrations, backups, and optional Back Office desktop app. Keep this PC on. |
| Register #1 | Windows PC | Primary cashier station. Owns the physical cash drawer, receipt printer, and Z-close for the till group. |
| Register #2 | iPad PWA | Satellite register lane. Must attach to the open Register #1 till group. |
| Other Back Office PCs | Browser/PWA or desktop app | Use for back-office work against the same server. Do not install the server role here. |
| Phones / remote devices | PWA over trusted network or Tailscale | Use only trusted/private access. Do not expose staff apps on public plain HTTP. |

Operational rule:

- Register #1 is the cash drawer lane.
- Register #2+ are satellite lanes.
- Z-close on Register #1 closes the till group.
- Physical cash lives in Register #1 even when a sale is entered from Register #2.

## 3. Before You Start

### Required

- [ ] Download and unzip `RiversideOS-v0.60.0-af23cef-Windows-Deployment.zip`.
- [ ] Keep the unzipped package on a USB drive or local folder available to both Windows PCs.
- [ ] Confirm you have Windows Administrator access on the Backoffice / Server PC.
- [ ] Confirm you have Windows Administrator access on Register #1.
- [ ] Confirm the store network is working.
- [ ] Confirm the Backoffice / Server PC has a stable name or IP address.
- [ ] Confirm Register #1 and iPad can reach the same network.
- [ ] Confirm Epson receipt printer IP or Windows printer name.
- [ ] Confirm Zebra scanner(s) are ready in HID keyboard mode.
- [ ] Confirm payment terminal device codes and provider credentials are available if using Helcim.
- [ ] Keep the prior installer and a fresh database backup available for rollback.

### Recommended

- [ ] Put the Backoffice / Server PC on UPS power.
- [ ] Reserve static/DHCP IPs for the Backoffice / Server PC and Epson receipt printer.
- [ ] Record every password, station role, device IP, and install result in the deployment log.

## 4. Backoffice / Server PC Install

Run this first.

### 4.1 Open Deployment Manager

1. On the Backoffice / Server PC, open the unzipped deployment package.
2. Right-click or double-click:

```text
Start-RiversideDeployment.cmd
```

3. Choose:

```text
Backoffice / Server
```

4. Click **Check**.
5. Resolve any failed readiness checks before installing.
6. Click **Install**.

The installer should:

- Install or use PostgreSQL.
- Create/update Riverside database credentials.
- Copy `riverside-server.exe`.
- Copy the web/PWA files.
- Apply migrations.
- Write `C:\RiversideOS\server\.env`.
- Add the firewall rule.
- Create startup scheduled task `Riverside OS Server`.
- Start the server.
- Check the local app URL.

### 4.2 PostgreSQL Password Prompts

- If PostgreSQL already exists, enter the existing `postgres` admin password.
- If the Deployment Manager installs PostgreSQL and password fields are blank, it can generate passwords.
- Do not commit or publicly share `riverside-deployment.config.json`.
- Keep the generated credentials with the store deployment records.

### 4.3 Server Acceptance Check

On the Backoffice / Server PC:

- [ ] Riverside opens locally.
- [ ] Staff sign-in screen loads.
- [ ] Settings -> Updates shows Riverside version `0.60.0`.
- [ ] Settings -> General -> About this build shows the expected server/API.
- [ ] Settings -> Data & Backups opens.
- [ ] Backup path is writable.
- [ ] Server restart works through the Deployment Manager or scheduled task.

From another device on the store network:

- [ ] Open the server URL, for example `http://SERVER-PC-NAME:3000`.
- [ ] Staff sign-in screen loads.
- [ ] No certificate/browser warning appears if using HTTPS.

### 4.4 Meilisearch / Help / ROSIE

After the server is installed:

- [ ] If Meilisearch is enabled, go to Settings -> Integrations -> Meilisearch.
- [ ] Run **Rebuild search index** after deploy, restore, Meilisearch wipe, or major import.
- [ ] Confirm product/customer/order search returns expected matches.
- [ ] Confirm Help Center opens.
- [ ] Confirm ROSIE is available only if the local model/runtime is installed and configured.

## 5. Register #1 Windows PC Install

Run this after the Backoffice / Server PC is working.

### 5.1 Open Deployment Manager

1. On Register #1, open the same unzipped deployment package.
2. Run:

```text
Start-RiversideDeployment.cmd
```

3. Choose:

```text
Register #1
```

4. Enter or confirm the API base for the Backoffice / Server PC.

Examples:

```text
http://SERVER-PC-NAME:3000
http://192.168.1.25:3000
https://ros.yourstore.example
```

Do not use `127.0.0.1` on Register #1 unless the Riverside server is actually running on Register #1.

5. Click **Check**.
6. Resolve failed checks.
7. Click **Install**.

The installer should:

- Write `C:\ProgramData\RiversideOS\station-config.json`.
- Install the Riverside desktop app from the package.
- Save the API base and printer settings.
- Launch the app when possible.

### 5.2 Register #1 First Launch

- [ ] Open the Riverside desktop app.
- [ ] Confirm the app icon is the Riverside logo, not an old placeholder.
- [ ] Confirm Settings -> Updates shows `0.60.0`.
- [ ] Confirm Settings -> General -> About this build shows the Register #1 API base.
- [ ] Staff sign-in works.
- [ ] Open Register #1.
- [ ] Enter the opening float according to store procedure.
- [ ] Confirm Register #1 is the cash drawer lane.

### 5.3 Receipt Printer And Cash Drawer

Preferred receipt printer setup:

```text
Mode: Network address
Printer: Epson TM-m30III
Port: 9100
IP: static or DHCP-reserved Epson IP
```

Use **Installed printer on this PC** only for USB/driver-managed fallback.

Checklist:

- [ ] Epson receipt printer has paper.
- [ ] Epson printer IP or Windows printer name is recorded.
- [ ] Printers & Scanners settings are saved.
- [ ] Test receipt prints.
- [ ] Cash drawer opens for CASH/CHECK tender.
- [ ] Cash drawer does not intentionally open for card-only tender or receipt reprint.
- [ ] Register #1 receipt reprint path works.

### 5.4 Scanner

- [ ] Scanner is in HID keyboard mode.
- [ ] Scanner sends Enter/Tab suffix if desired.
- [ ] POS product search receives scanned text.
- [ ] Inventory/search input receives scanned text.
- [ ] Charging/spare scanner plan is known.

### 5.5 Register #1 Smoke Test

Run one supervised safe flow:

- [ ] Sign in as staff.
- [ ] Open Register #1.
- [ ] Search or scan an item.
- [ ] Add item to cart.
- [ ] Open checkout drawer.
- [ ] Cancel safely or complete a store-approved low-risk test sale.
- [ ] Confirm receipt behavior.
- [ ] Confirm drawer behavior.
- [ ] Confirm Register #1 can Z-close the till group.

## 6. Register #2 iPad PWA Install

Register #2 is a PWA satellite lane. It should use the Backoffice / Server PC URL while inside the store.

### 6.1 Prepare iPad

- [ ] iPad is on the same store Wi-Fi/network as the Backoffice / Server PC.
- [ ] iPad date/time is correct.
- [ ] Safari is available.
- [ ] Bluetooth scanner, if used, is paired as keyboard/HID input.

### 6.2 Install PWA On iPad

1. Open Safari.
2. Visit the Riverside server URL.

Examples:

```text
http://SERVER-PC-NAME:3000
http://192.168.1.25:3000
https://ros.yourstore.example
```

3. Confirm the Riverside sign-in screen loads.
4. Tap **Share**.
5. Tap **Add to Home Screen**.
6. Name it:

```text
Riverside Register 2
```

7. Launch Riverside from the new Home Screen icon.

Do not run Register #2 from a regular Safari tab for production use.

### 6.3 Register #2 First Launch

- [ ] Launch from Home Screen icon.
- [ ] Staff sign-in works.
- [ ] Confirm Settings -> Updates shows version `0.60.0`.
- [ ] Confirm no **Update incomplete** warning.
- [ ] Open/attach Register #2 only after Register #1 is open.
- [ ] Confirm it behaves as a satellite lane.
- [ ] Confirm customer lookup works.
- [ ] Confirm item lookup/cart works.
- [ ] Confirm checkout drawer fits comfortably on the iPad.
- [ ] Confirm staff understand physical cash is handled at Register #1.

### 6.4 iPad Scanner

- [ ] Pair Zebra CS6080 or similar as Bluetooth keyboard/HID.
- [ ] Open a product/SKU search field.
- [ ] Scan a barcode.
- [ ] Confirm scanned text appears in the focused field.
- [ ] Confirm suffix behavior is acceptable.

### 6.5 iPad Receipts

Operational rule:

- Register #1 remains the primary receipt and cash-drawer station.
- iPad receipt printing is allowed only if the server can reach the Epson network printer.
- If iPad print fails, reprint from Register #1.

Checklist:

- [ ] If iPad receipt printing is expected, Epson printer has a network IP.
- [ ] Backoffice / Server PC can reach Epson printer IP on port `9100`.
- [ ] Server-side receipt print is tested from the iPad flow.
- [ ] Staff know the Register #1 reprint recovery path.

## 8. Updates After Install

After any update:

- [ ] Update Backoffice / Server PC first.
- [ ] Confirm server starts and Settings -> Updates shows `0.60.0`.
- [ ] Update Register #1 desktop app.
- [ ] Relaunch Register #1 and confirm Settings -> Updates shows `0.60.0`.
- [ ] On iPad, open Settings -> Updates -> iPad and browser app -> Check app files.
- [ ] If the PWA update prompt appears, tap **Reload now** only when no sale is in progress.
- [ ] Confirm no station shows **Update incomplete**.

Never run sales if server, Windows app, and PWA files show different Riverside versions.

## 9. Backup And Rollback Readiness

Before live use:

- [ ] Create a fresh database backup.
- [ ] Confirm the backup file exists and has non-trivial size.
- [ ] Keep a copy off the server PC if practical.
- [ ] Keep the previous installer/build accessible.
- [ ] Record backup timestamp and file name.
- [ ] Record who has rollback authority.

Rollback rule:

- Stop all registers.
- Close active transactions.
- Restore the database backup before running older binaries against migrated data.
- Do not run an older server against a newer migrated database without restore approval.

## 11. Quick Troubleshooting

### PWA will not load

1. Confirm the device is on the correct network.
2. Open the API/server URL in the device browser.
3. Check DNS/server name.
4. Check firewall on Backoffice / Server PC.
5. If using HTTPS, check certificate/date/time.
6. Close and reopen the Home Screen icon.
7. If still stale, remove the icon, clear site data, and Add to Home Screen again.

### Register app will not reach server

1. Check Settings -> General -> About this build for API base.
2. Confirm Register #1 is not pointed at `127.0.0.1` unless the server is local.
3. Ping/open the Backoffice / Server PC URL from Register #1.
4. Run Repair from Deployment Manager if station config is wrong.

### Receipt will not print

1. Check Printers & Scanners.
2. For network mode, confirm Epson IP and port `9100`.
3. For installed-printer mode, confirm Windows test page works.
4. Restart the desktop app after printer/network changes.
5. For iPad, confirm the server can reach the network printer.

### Update incomplete

1. Finish updating the Backoffice / Server PC.
2. Finish updating Register #1 desktop app.
3. On iPad/PWA, Check app files and reload.
4. Do not continue production work until all surfaces show `0.60.0`.

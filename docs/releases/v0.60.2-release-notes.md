# Riverside OS v0.60.2 — Modernized Deployment Manager + Administrative Utilities

Riverside OS v0.60.2 is a targeted release focused on modernizing the installation, update, and repair processes. It replaces the legacy PowerShell/WinForms-based deployment manager with a high-fidelity, interactive React + Tauri desktop application using the WowDash design system. It also integrates all previous command-line diagnostic utilities into a single, cohesive pane of glass.

## What changed

### Interactive Deployment Manager (v0.60.2)
- **Station Role Wizard**: Added a clean, step-by-step user interface to guide operators through setting up a Backoffice / Server PC or a Front Register station.
- **Config Form Validation**: Form-guided entries for Server IP address and PostgreSQL administrator password, with automatic JSON configuration serialization.
- **Color-Coded Terminal Logs**: A real-time, scrolling console viewer that streams output directly from the underlying PowerShell installation scripts (`install-server.ps1` and `install-register.ps1`), showing errors and successes immediately.
- **Port Isolation**: Configured the manager's development server to run on port `5179`, resolving port conflicts with the main POS development port (`5173`) that previously caused the manager app to load the main POS UI.

### Integrated Maintenance & Repair Panel
- **Server Control Center**:
  - **Start Server**: Directly starts the Windows Scheduled Task for the core API server.
  - **Restart Server**: Safely stops the task, forces any hanging `riverside-server` instances to close, and restarts the task cleanly.
  - **Open Logs**: Natively opens the server logs directory (`C:\RiversideOS\logs`) in Windows File Explorer.
  - **Check Package**: Validates the local deployment directory to ensure all binaries, migrations, and package manifests are present and uncorrupted.
- **Database & Migrations Workspace**:
  - **Apply Migrations**: Runs any pending database schema updates (`apply-riverside-migrations.ps1`) without restarting the server or dropping tables.
  - **Seed Database**: Runs an inline PowerShell command to search for `server\seed.sql` and execute it via `psql`.
  - **Factory Reset Database**: Triggers the `reset-riverside-database.ps1` script to drop, recreate, and migrate the DB schema, protected by a browser-native confirmation prompt to prevent accidents.
- **Utility Scripts**:
  - **Force ROSIE AI Update**: Downloads Gemma models and builds voice runtimes on demand.
  - **Sync Counterpoint Bridge**: Regenerates and maps CP Bridge tokens using the `set-counterpoint-bridge-token.ps1` utility.
  - **Repair Server Credentials**: Regenerates JWT encryption and signing keys.
  - **Repair Admin Account**: Bootstraps the default master administrator credentials if locked out.

## Deployment notes

- Backoffice / Server should use `http://127.0.0.1:3000` in the Windows desktop app.
- Register #1 and PWA devices should use the Backoffice / Server LAN or Tailscale URL, for example `http://10.64.70.196:3000`.
- All deployment management should now be handled through the new `RiversideOS-Deployment-Manager.exe` located in the deployment folder.
- Windows updater assets must be produced by the signed Tauri updater workflow so `latest.json`, the MSI, and the `.sig` all point to `0.60.2`.

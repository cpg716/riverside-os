# Riverside OS v0.60.0 — Production Release

## Overview

Riverside OS v0.60.0 is the production release that carries the v0.50 GOLD certification baseline into the final store deployment track. It focuses on release alignment, operational resilience, financial trust, Help/ROSIE recertification, Windows updater readiness, and production recovery procedures.

This release does not introduce speculative product scope. It packages the recent RC hardening already merged to `main` and aligns the runtime, desktop, PWA, server, Help Center, and deployment surfaces to one release identity.

## Highlights

- Register close and operational-readiness hardening for live store pressure.
- QBO/reporting updates for clearer accounting review and support handoff.
- Podium inbox, provider sync, and communication-status hardening.
- Help Center and staff documentation recertification for current workflows.
- Windows desktop recovery, updater, and deployment-package alignment.
- Staff bug-report exports labeled as AI diagnostic packages for Codex-ready support handoff.

## Deployment Notes

- Use the `v0.60.0` GitHub release assets only; do not mix older installer/updater assets with this server or PWA build.
- Confirm **Settings -> Updates** shows one expected Riverside version across the Windows app, server API, and PWA/web files.
- Keep the previous installer and a fresh pre-update database backup available for rollback.
- Run Help/ROSIE reindex after deployment when `docs/staff/**` or the Help corpus changes.

## Rollback

Rollback requires the database backup taken before deployment plus the matching previous server binary, web bundle, and Windows installer. Do not run an older binary against a database that has already received newer migrations without restoring the database.

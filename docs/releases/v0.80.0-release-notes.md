# Riverside OS v0.80.0 — SRE Production Hardening, Key Preservation, Directory ACLs, and Epson Pre-flight Warning

Riverside OS v0.80.0 is our "Gold" Production Master release, focused on SRE-grade operational security, deployment idempotency, and retail hardware verification logic to ensure flawless daily store operations.

## What changed

### 1. Client Cache & Register State Protection
* **Targeted Cleanup**: Modified `install-register.ps1` to prevent wiping the registry station client's WebView2 browser cache folder (IndexedDB, localStorage) on re-installs or updates. This safeguards offline transaction queues, custom printer selections, register bindings, and temporary local cashier sessions.

### 2. Encryption Key Preservation on Updates
* **Zero-Downtime Key Salvaging**: Modified `install-server.ps1` to check for an existing `RIVERSIDE_CREDENTIALS_KEY` in `server/.env`.
* If a key is present, the installer preserves it during updates instead of writing a new one. This prevents silent decryption failures for downstream integration credentials (Helcim, QBO, Counterpoint) stored in the PostgreSQL database.

### 3. Atomic Database Migrations
* **Power-Loss Recovery**: Appended the transaction wrap-around `-1` (single-transaction) flag to the `psql` command execution arguments in `install-server.ps1` and `apply-riverside-migrations.ps1`.
* If a migration is interrupted mid-execution (e.g., due to power loss), PostgreSQL rolls back the entire script atomically, leaving the database schema uncorrupted. On reboot, the server safely re-attempts the migration.

### 4. Process-Level Environments & Locked Directory ACLs
* **Hardened Security Boundaries**: Replaced all machine-level system environment variable writes with secure, process-level environment values populated directly inside startup tasks and process configurations.
* **Directory ACLs**: Locked down the root `C:\RiversideOS` directory permissions in `install-server.ps1` to `SYSTEM` and members of the `Administrators` local group. Unprivileged local users or processes can no longer read database credentials or app secrets in `server/.env`.

### 5. Epson Printer Pre-flight Diagnostics
* **Driver Validation**: Added a driver registration check in `Start-RiversideDeployment.ps1` that scans for Epson APD v6.07 drivers when the operator selects "Installed printer" mode. A warning is logged if the required driver is missing, preventing silent receipt printing failures.

## Deployment notes

* All versions synchronized at `0.80.0` across root, client, server, Tauri, ros-dev, and Deployment Manager manifests.
* Windows deployment builds must use `build-deployment-package.ps1` at tag `v0.80.0`.
* Ensure target servers have applied all database migrations. Rerunning updates is safe and fully idempotent.
* Root directory permissions will be automatically locked to `SYSTEM` and `Administrators` during `install-server.ps1` execution.

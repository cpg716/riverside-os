Alignment notes for schema probe output.
Current bridge behavior is runtime mapping for Riverside's NCR Counterpoint POS v8.4 environment.

Finding (report)                          →  Runtime mapping behavior
────────────────────────────────────────────────────────────────────────────
PS_SLS_REP, PO_BUYER not visible          →  SY_USR imports users; distinct SLS_REP values from AR_CUST + PS_TKT_HIST can post as sales-rep stubs.
PS_TKT_HIST_CELL / PS_TKT_HIST_LIN_CELL  →  Either satisfies “ticket matrix cells” in discover 0.6.10+. v8-style builds often have PS_TKT_HIST_LIN_CELL only.
                                            Missing ticket-cell mapping is OK. Server ingest may still resolve matrix SKUs
                                            when the line has parent ITEM_NO only and unit price matches one matrix variant’s retail.
PS_TKT_HIST_GFT not visible              →  Historical gift tender details are skipped; current gift-card balances still map from visible gift-card masters.
SY_GFT_CERT missing but SY_GFC present   →  Runtime mapper can use the visible gift-card master table when supported columns are present.
SY_GFT_CERT / SY_GFT_CERT_HIST missing   →  Gift-card current balances are skipped when no supported card master is visible.
PS_LOY_PTS_HIST missing                  →  Historical loyalty activity remains off; current loyalty points map from AR_CUST when a balance column is visible.
IM_INV LST_COST present                  →  Inventory/catalog cost mapping uses the visible cost column.
AR_CUST + PS_TKT_HIST / AR_CUST_NOTE     →  Customers, notes, store credit, and loyalty balance mappings are generated from visible columns.
IM_ITEM / IM_INV / IM_INV_CELL / tickets →  Catalog, vendors, vendor item links, cells, ticket headers, lines, and payments are generated from visible columns.
LOY_PTS_BAL on AR_CUST                   →  Customer mapping uses LOY_PTS_BAL as loyalty_points.
TOT_EXTD_PRC and TOT_AMT_DUE absent      →  Ticket mapping can fall back to TOT when those columns are not visible.
PO_VEND_ITEM UNIT_COST present           →  Vendor-item mapping uses UNIT_COST as vend_cost.

If your live .env was copied from an older template, remove stale CP_*_QUERY values. Normal setup keeps only ROS_BASE_URL and SQL_CONNECTION_STRING.

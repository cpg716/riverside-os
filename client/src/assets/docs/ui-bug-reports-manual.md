# Reporting Bugs & Technical Feedback

## Role: All Staff
### Purpose: Documenting system issues for administrative and engineering triage.

If Riverside OS behaves unexpectedly, crashes, or displays incorrect data, use the **Report a Bug** feature to notify IT and management.

---

## What a Bug Report Includes

When you submit a report, the system automatically packages:
- **Screenshot**: A snapshot of your current screen (can be disabled).
- **Diagnostics**: Browser version, active tab, and current session ID.
- **Server Trace**: A "log slice" of recent activity to help engineers see exactly what failed.
- **Your Summary**: A description of the problem and the steps taken to reach it.

---

## How to Submit a Report

1. **The Bug Icon**: Tap the **Bug** icon in the **Global Top Bar** (Back Office or POS).
2. **Snapshot**: By default, a screenshot of the main workspace is included. **Uncheck** this if sensitive customer payment data (like a card number being keyed in) is visible.
3. **The Form**:
   - **What went wrong?**: Describe the error (e.g., "The 'Save' button is unresponsive on the Customer Profile").
   - **What were you doing?**: List the steps (e.g., "Opened Relationship Hub > Measurements > Edited Waist > Tapped Save").
4. **Submit**: Wait for the "Success" toast. A unique **Reference ID** will be provided for your tracking.

---

## Guidelines for Effective Reports

- **Stay on Screen**: If the bug is visual, open the reporter while the error is still visible on screen.
- **One Bug per Report**: Avoid grouping unrelated issues.
- **Urgency**: Bug reports are for **non-critical** triage. For emergency outages (e.g., cannot process payments), follow your store's immediate escalation SOP (Call Manager/IT).

---

## Common Messages

| Message | Meaning |
| :--- | :--- |
| **Too many reports** | Rate limit reached (Max 12 reports per 15 mins per staff). Wait and try again. |
| **Describe the issue** | Both text fields must be filled before the system can accept the report. |
| **Network Error** | The server is unreachable. Document the issue manually and notify a manager. |

> [!NOTE]
> Bug reports are stored internally and reviewed by the store's Admin and the Riverside OS dev team. They are never visible to customers.
